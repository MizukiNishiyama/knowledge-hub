## 2. 勝ち筋A: 災害対応WiFiセンシング — 詳細評価

### 2.1 判定: No-Go

RuViewのWiFi-Mat機能は、マーケティング資料として魅力的だが、**技術的に成立しない**。

### 2.2 RuView主張 vs 物理的現実

| RuViewの主張 | 査読論文の限界値 | ギャップ |
|---|---|---|
| 壁越し検知距離: 15-40m | 標準アンテナ: **2-4m**、指向性アンテナ（ラボ）: 最大18.5m | 主張値は物理限界の2-10倍 |
| コンクリート越し: 5-10m | 学術研究結論:「NLOSシナリオではCSI測定で十分な信頼性が得られない」 | コンクリート越しバイタル検出は学術的に否定的 |
| 瓦礫越し生存者検知 | WiFi CSIで瓦礫越し検知を行った査読論文: **ゼロ** | 学術的前例すら存在しない |
| START トリアージ分類 | 瓦礫越しCSIデータでのトリアージ検証: **ゼロ** | 全テストが合成信号 |
| 実災害での利用 | WiFi CSIの実災害利用事例: **世界でゼロ** | 技術成熟度が根本的に不足 |

### 2.3 実際に災害現場で使われている技術

| 技術 | 製品例 | 実績 | 検知距離 |
|---|---|---|---|
| **UWBレーダー** | SJ-6000+, Xaver 400 | トルコ2023、四川2008、メキシコ2017 | 42cm壁越し18-27m |
| 音響探知 | Delsar, Zircon SearchCam | 多数の地震で標準使用 | 瓦礫越し数m〜 |
| ドップラーレーダー | NASA FINDER | ネパール地震2015 | コンクリート6m越し |
| WiFi CSI | なし | **ゼロ** | 壁越し2-4m（ラボ） |

### 2.4 災害対応の結論

WiFi CSIは動体検知・在室検知には有効だが、**瓦礫・コンクリート越しの生存者探知は現在の技術水準では不可能**。この用途にはUWBレーダーが確立された実用技術であり、WiFi CSIが優位性を持つ物理的根拠がない。RuViewのWiFi-Matは実証データを伴わないマーケティング主張であり、事業の基盤にはできない。

## 3. 勝ち筋B: 高齢者見守りプレミアム — 詳細評価

### 3.1 判定: 条件付きGo（RuViewは使わず、学術手法ベースで再設計）

WiFi CSIによるバイタルセンシングは学術的に実証されている。ただしRuView自体ではなく、**査読済みの手法（PulseFi等）をベースに独自実装**する前提。

### 3.2 学術的に検証された精度ベンチマーク

#### PulseFi（UC Santa Cruz, IEEE DCOSS-IoT 2025）— 最も信頼できるベンチマーク

| 指標 | PulseFi結果（ESP32実測） | 条件 |
|---|---|---|
| 心拍 MAE | **0.50 BPM** | ESP32-S3、5秒窓、静止・単独被験者 |
| 呼吸 MAE | **0.09 BPM** | ESP32-S3、20秒窓、静止・単独被験者 |
| 心拍 臨床閾値内率 | **97.95%** | 臨床基準±5 BPM以内 |
| 被験者数 | **118人**（ESP32: 7人、EHealth: 118人） | 多様な体型・年齢 |

#### 精度の現実的限界

| 条件 | 呼吸検出精度 | 心拍検出精度 | 備考 |
|---|---|---|---|
| 静止・単独・LoS | 95-98% | 90-97% | 最良条件 |
| 静止・2人同時 | 85-92% | 75-85% | 干渉が発生 |
| 静止・5人同時 | **62%** | **推定50%以下** | 多人数分離は未解決の研究課題 |
| 軽い体動あり | 70-80% | 50-70% | 体動が信号を支配 |
| 壁越し（NLoS） | 60-75% | **不安定** | 学術的に「信頼性不十分」 |

### 3.3 高齢者見守りへの適用可能性の段階的評価

| 機能 | 技術成熟度 | 高齢者見守りでの実用性 | 推奨 |
|---|---|---|---|
| **在室検知**（人がいるかいないか） | ★★★★★ 商用実証済み | 即座に提供可能。MAMOLEO等で実績あり | **Phase 1で提供** |
| **活動監視**（動いているか止まっているか） | ★★★★☆ 高精度 | 異常検知（長時間不動）に有効。転倒検知も研究段階 | **Phase 1で提供** |
| **呼吸検知**（呼吸しているか） | ★★★☆☆ 条件付き | 静止・単独なら高精度。ただし高齢者特有の低呼吸・不規則呼吸は未検証 | **Phase 2で追加**（PoC検証後） |
| **心拍検知** | ★★☆☆☆ 実験段階 | 静止・単独でMAE 0.5 BPM。高齢者の不整脈対応は未検証。臨床利用不可 | **Phase 3で研究開発** |
| **姿勢推定** | ★☆☆☆☆ 未実証 | RuViewの17点姿勢推定は実環境で未動作。学術的にも商用品質に至っていない | **スコープ外** |

### 3.4 推奨プロダクトアーキテクチャ

```
Phase 1（MVP — 3ヶ月）
├── 在室検知: Espressif ESP-CSI公式ツールキットベース

├── アラート: 長時間不動検知 → 家族アプリ通知

├── 精度検証: 高齢者10人での実環境テスト
└── 競合優位: MAMOLEO比で「呼吸見守り」が差別化要因

Phase 3（研究開発 — 12ヶ月〜）
├── 心拍検知: PulseFi拡張（自己相関 + 位相コヒーレンス）
```

### 3.5 技術選定の推奨

## 4. WiFi CSIセンシング技術の全体評価

### 4.1 技術は本物か？

**WiFi CSIによるセンシング技術自体は本物。** 10年以上の学術研究の蓄積があり、以下は査読論文で実証済み:

| 機能 | 学術的実証 | 商用実績 |
|---|---|---|
| 動体検知（人がいるか） | 多数の論文で99%超 | Origin AI、MAMOLEO、Linksys Aware等で商用化済み |
| 在室検知（部屋にいるか） | 高精度で実証済み | Origin AI TruPresence等 |
| 呼吸検知 | PulseFi等でMAE 0.09 BPM | **商用製品なし**（精度保証の困難さ） |
| 心拍検知 | PulseFi等でMAE 0.50 BPM | **商用製品なし**（環境依存性が高い） |
| 壁越し検知 | 薄い壁なら可能（2-4m） | **商用製品なし**（コンクリートで急激に劣化） |
| 姿勢推定 | 限定的な研究あり | **商用製品なし**（精度不足） |

### 4.2 RuViewは本物か？

**RuViewのアルゴリズム設計は学術的知見に基づいているが、実装は未完成・未検証であり、プロダクトの基盤としては信頼できない。**

「RuViewの技術」と「WiFi CSIの技術」は別物。後者は本物、前者は疑問あり。

## 5. 最終結論と推奨アクション

### 判断

| 項目 | 結論 |
|---|---|
| RuViewをプロダクト基盤として使用 | **No** |
| WiFi CSI技術で高齢者見守り事業を構築 | **条件付きYes** |

### 推奨ネクストアクション（優先順）

- [ ] **ESP32-S3 + Espressif ESP-CSI公式ツールキットで在室検知PoC構築**（1-2週間）。RuViewは使わない。CSIデータの取得と基本的な動体検知をまず動作確認
- [ ] **PulseFi論文の精読と呼吸検知アルゴリズムの再実装**（2-4週間）。IEEE DCOSS-IoT 2025論文の手法をESP32-S3で再現し、自宅環境で精度を測定
- [ ] **介護施設3箇所へのヒアリング**（2週間）。「カメラなし・装着なしの見守り」への需要と、月額1,500-5,000円の価格感覚を検証
- [ ] **MAMOLEO / ai6社の詳細調査**。技術スタック・特許状況・今後のバイタル機能追加計画を把握。競合か協業かを判断

### 本アイディアの位置づけ（ポートフォリオ内）

000_AI人流分析プロジェクトとの技術的シナジーは限定的（WiFi CSI vs カメラ+視線トラッキングで技術スタックが異なる）だが、「非接触センシング」「プライバシー保護」というコンセプトレベルでの親和性はある。高齢者見守り事業が軌道に乗った場合、WiFiセンシングの人流分析応用（勝ち筋C）への横展開は中長期的に検討可能。
