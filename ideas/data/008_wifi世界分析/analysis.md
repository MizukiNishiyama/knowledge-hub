## 結論（条件付きGo — ただし「WiFiセンシングプラットフォーム」ではなく「特定ドメインのソリューション」として参入せよ）

WiFiセンシング技術は、ADTによるOrigin AI買収（$170M）で事業価値が証明された。しかし、**基本機能（動体検知・在不在判定）は既にISP・セキュリティ大手がコモディティ化を進めており、「WiFiセンシングプラットフォーム」として参入するタイミングは遅い**。一方、RuViewのOSSを活用した「バイタルセンシング（呼吸・心拍）」「壁越し検知」「姿勢推定」は、**いまだ商用化されていない先端領域**。この先端機能を、特定ドメイン（災害対応 or 高齢者見守りのプレミアム層）に特化して提供するなら勝機がある。ただし、RuViewの実環境精度に課題報告があり、OSSコードの検証が最優先。

## 勝ち筋の候補

### 勝ち筋A: 災害対応WiFiセンシング（推奨 — 高単価・高インパクト）

**なぜこの切り口が有効か:**
- RuViewの最も独自性の高い機能「WiFi-Mat」（瓦礫越し生存者検知、START triage分類、3D位置推定）は、**競合が一切存在しない唯一のユースケース**
- 日本は地震大国。2024年能登半島地震以降、自治体・消防の災害テック投資が加速。「72時間の壁」（生存率が急落する時間制限）を突破するツールに対する切実なニーズ
- ESP32ノード4台で$48（約7,200円）という圧倒的な低コストで、大量配備が可能。従来の生存者探査装置（音響・赤外線・レーダー）は数百万〜数千万円
- 防衛省・消防庁・JICA等の公的調達ルートで高単価が狙える。1セット10-50万円でも安い
- 地震国（日本・台湾・トルコ・インドネシア等）への水平展開が可能

**ビジネスモデル:**
- 誰に: 消防庁・自治体消防・自衛隊・JICA・国際赤十字・NGO
- 何を: ESP32ベースのポータブル生存者探査キット（WiFi-Mat）。瓦礫越しの人体検知・トリアージ分類・3D位置推定
- 課金: ハードウェアキット販売（1セット10-50万円）+ 年間保守/アップデート契約（キット価格の15-20%）
- 初期: 防災科学技術研究所 or 消防研究センターとの共同実証実験

**リスク・前提条件:**
- **最大のリスク: WiFi-Matの実環境精度が商用品質に達しているか未検証**。瓦礫越し5mの検知精度、コンクリート・鉄筋での減衰、ノイズ環境での安定性を実測する必要がある
- 公的調達は意思決定が遅い（1-3年のリードタイム）。キャッシュフローが厳しい
- 災害は「いつ来るかわからない」ため、平時の収益源が必要（訓練・デモ・保守で補完）
- 軍事転用リスクの管理。輸出管理規制への対応が必要

**なぜ今:** 能登半島地震（2024年1月）で災害テックへの社会的関心が最高潮。南海トラフ地震の発生確率が上がり続けている。ESP32-S3の価格低下でハードウェアコストが実用的になった。RuViewのOSSリリースにより、ゼロからの研究開発が不要になった。

### 勝ち筋B: 高齢者見守りプレミアム（バイタルセンシング付き）

**なぜこの切り口が有効か:**
- 既存の見守りサービス（センサー型・カメラ型）は「動いているか動いていないか」の二値判定。WiFiバイタルセンシングで「呼吸・心拍の異常検知」まで踏み込めば、**死角のない非接触見守り**が実現
- 高齢者のウェアラブル装着率は極めて低い（充電忘れ・装着拒否）。WiFiセンシングは完全パッシブで装着不要
- MAMOLEO（月額1,100円）が動体検知のみに対し、バイタルセンシング付きで月額3,000-5,000円のプレミアム帯を狙える
- 高齢者見守り市場は227億円（2025年予測）で安定成長。後期高齢者の増加で市場拡大が確実

**ビジネスモデル:**
- 誰に: 独居高齢者の家族、介護事業者、自治体（地域包括ケア）
- 何を: ESP32センサーキット + クラウドダッシュボード + 家族向けアプリ。呼吸・心拍の異常時にアラート
- 課金: ハードウェアキット（初期1-2万円）+ 月額3,000-5,000円（クラウド分析+アラート）
- 初期: 介護施設2-3箇所でのPoC → 自治体の見守り事業の実証実験に応募

**リスク・前提条件:**
- **バイタルセンシングの精度**: RuViewのバイタル推定は「heuristic BPM extraction」であり、医療品質ではない。安静時かつ制御環境でのみ精度が出る可能性。実環境（テレビつけっぱなし、エアコン稼働、ペットがいる等）での精度検証が必須
- **薬機法の壁**: 「呼吸・心拍を測定する」と謳うと医療機器に該当する可能性。「活動量の目安」として提供するか、医療機器認証を取得するかの判断が必要
- **MAMOLEO / ai6の先行**: 日本市場でWiFiセンシングの商用化実績があるai6社が、バイタル機能を追加してきた場合の競合リスク
- **Amazonの参入リスク**: Amazon eeroがWiFiセンシング+高齢者見守りを標準搭載した場合、価格で勝てない

**なぜ今:** 日本の後期高齢者人口が2025年に2,000万人を突破し、独居高齢者の孤独死が社会問題化。MAMOLEO が日本初のWiFiセンシング商用化を果たし、市場教育コストが下がりつつある。

### 勝ち筋C: OOH/施設向け匿名人流分析（000_AI人流分析との接続）

**なぜこの切り口が有効か:**
- 000_AI人流分析プロジェクトで検討中の「AIカメラ×視線トラッキング」に対し、WiFiセンシングは「カメラなし」で人の存在・動き・滞留時間を検知できる補完技術
- 商業施設・イベント会場・駅構内などの人流分析において、カメラ設置が困難な場所（トイレ周辺・更衣室近辺等）でもWiFiセンシングなら対応可能
- GDPRやカメラ規制が厳しくなる中、「映像を一切使わない人流分析」はプライバシー訴求で差別化
- ESP32の低コストで大量センサー配備が可能。1フロアあたり数万円で全域カバー

**ビジネスモデル:**
- 誰に: 商業施設デベロッパー、イベント運営企業、交通機関
- 何を: WiFiセンサーネットワーク + 匿名人流ダッシュボード（滞留時間・動線・混雑度のリアルタイム可視化）
- 課金: センサー設置費（10-30万円/フロア）+ 月額5-10万円/施設（SaaS）
- 初期: 商業施設1箇所でのPoC

**リスク・前提条件:**
- 人流分析市場は既にカメラベース（ABEJA・RetailNext等）が確立。WiFiセンシングの精度（位置推定のsub-30mm精度が本当か）が劣る場合、乗り換え動機が弱い
- WiFi干渉が多い商業施設での精度劣化リスク
- 000_AI人流分析プロジェクトとのシナジーは大きいが、技術スタックが異なるため統合コストが発生

## 総合所見

### 市場の魅力度と参入タイミング
**市場は急成長しているが、参入ポジションの選択が極めて重要**。WiFiセンシングの基本機能（動体検知）はADT/Comcast/オプテージ等の大手が既にコモディティ化を進めており、ここに参入しても勝てない。一方、「バイタルセンシング」「壁越し検知」「姿勢推定」は誰も商用化していない先端領域であり、**OSSのRuViewを活用してここに一足飛びに参入できる**ことが最大のアドバンテージ。ただし、先端領域が商用化されていない理由が「技術が未成熟」である可能性を、自ら検証する必要がある。

### 技術的モートの有無
**短期的にはOSS活用でモートなし。中期的にはドメイン知識とデータでモートを築ける可能性あり。**
- RuViewはMIT Licenseのため、誰でも同じコードを使える。コード自体は差別化にならない
- ただし、**「RuViewの実環境チューニング」「特定ドメイン（災害/介護）向けのカスタマイズ」「実証データの蓄積」**は、先にやった者が有利。OSSを読んで動かすだけなら誰でもできるが、「瓦礫越しの精度を○%まで向上させた実績」は真似できない
- Origin AIの特許ポートフォリオが潜在的リスク。WiFiセンシングの基礎特許を侵害しないか確認が必要

### チームに求められるケイパビリティ
- **組込みエンジニア**: ESP32ファームウェア開発、CSIデータ処理、カスタムハードウェア設計
- **信号処理/ML エンジニア**: CSIデータからのバイタルサイン抽出、ノイズ除去、精度向上
- **ドメインエキスパート**: 災害対応（消防・自衛隊出身者）or 介護（介護事業者・ケアマネ）
- **ハードウェア量産経験者**: 中国EMSとの量産管理、技適認証取得

### 最大の懸念点
1. **RuViewの実環境精度**: README上のスペック（sub-30mm、壁越し5m、17点姿勢推定）と実環境での精度にギャップがある可能性が高い。GitHub Issueで「sensing page provides extremely inaccurate data」との報告あり。**プロダクト化する前に自ら検証することが絶対条件**
2. **IEEE 802.11bf規格の策定**: 2025年にWiFiセンシングの標準規格が策定されると、全てのWiFiデバイスがセンシング機能を持つ。これはプラットフォーム企業（Apple/Google/Amazon）にとって参入障壁を下げるが、独立プロダクトの存在意義を脅かす
3. **Origin AIの特許リスク**: WiFiセンシングの基礎特許をOrigin AI（現ADT傘下）が保有。商用化時に特許侵害の主張を受ける可能性
4. **悪用リスク**: 壁越し検知技術はストーキング・盗撮への悪用が容易に想像される。技術的・法的な悪用防止策がなければ、社会的批判を受ける
5. **ハードウェアスタートアップの難しさ**: ソフトウェアSaaSと異なり、在庫・品質管理・返品対応・故障対応が必要。初期資金とオペレーション負荷が大きい
