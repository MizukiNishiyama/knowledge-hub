## 結論（条件付きGo — ただしRuViewは使用せず、学術手法ベースで「高齢者見守り」に特化せよ）

RuView実態調査の結果、プロジェクトの信頼性に致命的な問題が判明した（スター操作疑惑、全テストが合成データ、フォークによるSCAM ALERT警告、WiFiアダプター未接続でも結果表示）。**RuViewをプロダクト基盤として使用することは推奨しない。** ただし、WiFi CSIによるセンシング技術自体は10年以上の学術実績があり本物。災害対応（勝ち筋A）は物理的限界によりNo-Go。高齢者見守り（勝ち筋B）は、**Espressif ESP-CSI公式ツールキット + PulseFi（IEEE査読済み）の手法をベースに独自実装**する前提で条件付きGo。

## 勝ち筋の候補

### 勝ち筋A: 災害対応WiFiセンシング（~~推奨~~ → No-Go に変更）

**RuView実態調査により棄却。理由:**

1. **WiFi CSIの物理的限界**: 査読論文での壁越し検知は標準アンテナで2-4m（指向性アンテナでもラボ環境で最大18.5m）。RuViewの主張する15-40mは物理限界の2-10倍であり非現実的
2. **コンクリート越し検知は学術的に否定的**: 学術研究の結論「NLOSシナリオではCSI測定で十分な信頼性が得られない」。瓦礫（コンクリート+鉄筋+土砂）越しは論外
3. **WiFi CSIの実災害利用事例は世界でゼロ**: 実際の災害で使われているのはUWBレーダー（SJ-6000+, Xaver 400）で、42cm壁越し18-27m検知の実績がある
4. **RuView WiFi-Matは非機能的**: 全テストが合成信号、3,536フォーク中で災害機能を検証したものゼロ、フォークからSCAM ALERT警告
5. **瓦礫越し生存者検知を扱った査読論文自体が存在しない**: WiFi CSIでこの用途は学術的前例すらない

### 勝ち筋B: 高齢者見守りプレミアム（条件付きGo維持 — RuView不使用、学術手法ベース）

**なぜこの切り口が有効か:**
- 既存の見守りサービス（センサー型・カメラ型）は「動いているか動いていないか」の二値判定。WiFiバイタルセンシングで「呼吸・心拍の異常検知」まで踏み込めば、**死角のない非接触見守り**が実現
- 高齢者のウェアラブル装着率は極めて低い（充電忘れ・装着拒否）。WiFiセンシングは完全パッシブで装着不要
- MAMOLEO（月額1,100円）が動体検知のみに対し、バイタルセンシング付きで月額3,000-5,000円のプレミアム帯を狙える
- PulseFi（IEEE DCOSS-IoT 2025, 118人検証）で呼吸MAE 0.09 BPM・心拍MAE 0.50 BPMが実証済み。**学術的に検証された手法が存在する**
- 高齢者見守り市場は227億円（2025年予測）で安定成長。後期高齢者の増加で市場拡大が確実

**ビジネスモデル（修正版）:**
- 誰に: 独居高齢者の家族、介護事業者、自治体（地域包括ケア）
- 何を: ESP32-S3センサーキット + クラウドダッシュボード + 家族向けアプリ
- Phase 1: 在室検知+活動監視（月額1,500-2,500円）→ Phase 2: 呼吸検知追加（月額3,000-5,000円）
- 技術基盤: **Espressif ESP-CSI公式ツールキット + PulseFi手法の独自実装**（RuViewは使用しない）
- 初期: 介護施設2-3箇所でのPoC → 自治体の見守り事業の実証実験に応募

**リスク・前提条件（調査結果を反映）:**
- **バイタルセンシングの精度限界**: PulseFi実測で心拍MAE 0.50 BPM（静止・単独・5秒窓）が上限。5人同時で呼吸精度96%→62%に低下。多人数分離は未解決の研究課題
- **高齢者特有の検証データが存在しない**: 不整脈、低呼吸、体動の多さへの対応は未検証。独自の臨床検証が必要
- **環境依存性が極めて高い**: 壁材質、家具配置、距離、ペット、エアコン気流で大幅に変動
- **薬機法の壁**: 「呼吸・心拍を測定する」と謳うと医療機器に該当する可能性。Phase 1は「活動見守り」、Phase 2は「ウェルネス目安」として提供
- **MAMOLEO / ai6の先行**: 日本市場でWiFiセンシングの商用化実績があるai6社が、バイタル機能を追加してきた場合の競合リスク
- **Amazonの参入リスク**: Amazon eeroがWiFiセンシング+高齢者見守りを標準搭載した場合、価格で勝てない

**なぜ今:** 日本の後期高齢者人口が2025年に2,000万人を突破し、独居高齢者の孤独死が社会問題化。MAMOLEO が日本初のWiFiセンシング商用化を果たし、市場教育コストが下がりつつある。PulseFiの論文公開（2025年）により、ESP32でのバイタル検出の学術的エビデンスが揃った。

### 勝ち筋C: OOH/施設向け匿名人流分析（000_AI人流分析との接続）

**なぜこの切り口が有効か:**
- 000_AI人流分析プロジェクトで検討中の「AIカメラ×視線トラッキング」に対し、WiFiセンシングは「カメラなし」で人の存在・動き・滞留時間を検知できる補完技術
- 商業施設・イベント会場・駅構内などの人流分析において、カメラ設置が困難な場所（トイレ周辺・更衣室近辺等）でもWiFiセンシングなら対応可能
- GDPRやカメラ規制が厳しくなる中、「映像を一切使わない人流分析」はプライバシー訴求で差別化
- ESP32の低コストで大量センサー配備が可能。1フロアあたり数万円で全域カバー

**ビジネスモデル:**
- 誰に: 商業施設デベロッパー、イベント運営企業、交通機関
- 何を: WiFiセンサーネットワーク + 匿名人流ダッシュボード（滞留時間・動線・混雑度のリアルタイム可視化）
- 課金: センサー設置費（10-30万円/フロア）+ 月額5-10万円/施設（SaaS）
- 初期: 商業施設1箇所でのPoC

**リスク・前提条件:**
- 人流分析市場は既にカメラベース（ABEJA・RetailNext等）が確立。WiFiセンシングの精度（位置推定のsub-30mm精度が本当か）が劣る場合、乗り換え動機が弱い
- WiFi干渉が多い商業施設での精度劣化リスク
- 000_AI人流分析プロジェクトとのシナジーは大きいが、技術スタックが異なるため統合コストが発生

## 総合所見

### 市場の魅力度と参入タイミング
**市場は急成長しているが、RuViewの実態調査により参入戦略を大幅修正。** WiFiセンシングの基本機能（動体検知）はADT/Comcast/オプテージ等の大手が既にコモディティ化。壁越し検知・姿勢推定は物理的・技術的限界により商用化不可能。**唯一の現実的な参入点は「バイタルセンシング（呼吸検知）付き高齢者見守り」**であり、PulseFi等の学術手法を基盤にPhase分けで段階的に展開するアプローチが妥当。

### 技術的モートの有無
**RuViewはモートにならない（プロジェクト自体の信頼性が低い）。モートを築くとすれば:**
- **高齢者ドメインの実証データ**: 「ESP32 CSIで高齢者の呼吸異常を検知した実証データ」は、先にやった者が有利。PulseFiは若年〜中年の被験者データのみで、高齢者固有のデータは存在しない
- **介護事業者・自治体とのパートナーシップ**: 見守り事業は信頼関係が重要。先行導入実績が後続の営業を加速
- Origin AIの特許ポートフォリオが潜在的リスク。WiFiセンシングの基礎特許を侵害しないか確認が必要

### チームに求められるケイパビリティ
- **組込みエンジニア**: ESP32ファームウェア開発、CSIデータ処理、カスタムハードウェア設計
- **信号処理/ML エンジニア**: CSIデータからのバイタルサイン抽出、ノイズ除去、精度向上
- **ドメインエキスパート**: 災害対応（消防・自衛隊出身者）or 介護（介護事業者・ケアマネ）
- **ハードウェア量産経験者**: 中国EMSとの量産管理、技適認証取得

### 最大の懸念点
1. **RuViewプロジェクトの信頼性崩壊**: 実態調査によりスター操作疑惑、合成データのみのテスト、SCAM ALERT警告、WiFiアダプター未接続での結果表示が判明。**RuViewはプロダクト基盤として使用不可**
2. **バイタルセンシングの環境依存性**: PulseFi実測でも静止・単独・制御環境が前提。高齢者の実生活環境（体動・ペット・エアコン等）での精度は未知数
3. **IEEE 802.11bf規格の策定**: 2025年にWiFiセンシングの標準規格が策定されると、全てのWiFiデバイスがセンシング機能を持つ。Apple/Google/Amazonの参入障壁が下がる
4. **Origin AIの特許リスク**: WiFiセンシングの基礎特許をOrigin AI（現ADT傘下）が保有。商用化時に特許侵害の主張を受ける可能性
5. **ハードウェアスタートアップの難しさ**: ソフトウェアSaaSと異なり、在庫・品質管理・返品対応・故障対応が必要。初期資金とオペレーション負荷が大きい
