# 日本の現代の政治家 — リサーチ実行リファレンス

## 目的

現職の衆議院議員・参議院議員の全員について、構造化されたプロファイルを作成する。
政治家の名前・顔・経歴・実績が紐づいていない状態を解消し、政策議論やニュース読解時に即座に文脈を引けるナレッジベースを構築する。

## ディレクトリ構成

```
ideas/data/011_日本の現代の政治家/
├── note.md                    （元メモ）
├── reference.md               （本ファイル）
├── 衆議院/
│   └── {政党名}/
│       └── {議員名}/
│           ├── summary.md     （プロファイル）
│           └── sources.md     （出典一覧）
└── 参議院/
    └── {政党名}/
        └── {議員名}/
            ├── summary.md
            └── sources.md
```

### 命名規約
- 政党名: 正式名称を使用（例: `自由民主党`、`立憲民主党`、`日本維新の会`、`公明党`、`日本共産党`、`国民民主党`、`れいわ新選組`、`参政党`、`社会民主党`、`無所属`）
- 議員名: フルネーム漢字（例: `石破茂`、`野田佳彦`）

---

## 実行手順

### Phase 1: 議員名簿の取得

1. 衆議院公式サイト（shugiin.go.jp）から現職議員の全一覧を取得
2. 参議院公式サイト（sangiin.go.jp）から現職議員の全一覧を取得
3. 政党別にディレクトリを作成

### Phase 2: 各議員のリサーチ・プロファイル作成

各議員について以下の情報をWeb検索で収集し、summary.md / sources.md を生成する。

**検索戦略**:
- 一次情報を優先: 国会議員公式サイト、議会公式DB、政党公式サイト
- 補助: Wikipedia、選挙ドットコム（go2senkyo.com）、政治山、各報道機関
- 不確実な情報は「〜と報じられている」と明記

## summary.md フォーマット（1議員あたり）

```markdown
# {議員名}

## 1. 基本情報

| 項目 | 内容 |
|------|------|
| 氏名 | （漢字・ふりがな） |
| 生年月日 | （YYYY-MM-DD） |
| 年齢 | （歳） |
| 院 | 衆議院 / 参議院 |
| 選挙区 | （例: 東京1区 / 比例東京ブロック / 神奈川県選挙区 / 比例代表） |
| 当選回数 | （回） |
| 政党 | （現所属） |
| 派閥・グループ | （該当がある場合） |

## 2. 経歴（時系列）

- YYYY: （学歴・職歴・政治キャリアを時系列で箇条書き）

## 3. 歴任ポスト

| 期間 | ポスト |
|------|--------|
| YYYY-YYYY | （大臣、副大臣、政務官、委員長、党役職等） |

## 4. 政策スタンス・専門領域

- （経済・外交・安全保障・社会保障・デジタル・エネルギー等、主要な政策領域とスタンスを2〜5項目）

## 5. 主な功績・実績

- （立法実績、政策実現、党内での役割等。客観的に記述）

## 6. 第三者評価・注目点

- （メディア評価、有識者の評価、支持率データ、スキャンダル・批判があれば併記）
- （AI・テクノロジー政策に関与している場合は特記）

## 7. 概要（30文字以内）

（端的に何者かを表す一文）
```

## sources.md フォーマット

```markdown
# ソース一覧: {議員名}

| # | タイトル | URL | 種別 | 概要 |
|---|---------|-----|------|------|
| 1 | ... | ... | 公式サイト/議会DB/報道/選挙DB | ... |
```

## 品質基準

- 基本情報（氏名・選挙区・政党・当選回数）は**必須**。欠損不可
- 経歴は最低限、最終学歴・主要な職歴・初当選年を含むこと
- 歴任ポストが無い場合（新人議員等）は「なし」と明記
- 政策スタンスは客観的に記述し、特定の立場からの評価を避ける
- 第三者評価は肯定・否定の両面を併記する
- AI・デジタル政策に関与している議員は **`## 6. 第三者評価`** 内で特記する（西山氏の関心軸との接点）
- 各議員につき最低2つ以上のソースを参照する
- 情報が見つからない項目は「情報なし」と明記（空欄にしない）

## 優先順位

全議員を網羅するが、リサーチの深度に以下の優先順位をつける:

### Tier 1（詳細リサーチ）
- 現職閣僚（内閣総理大臣・国務大臣）
- 党首・幹事長クラス（主要政党の執行部）
- AI・デジタル・科学技術政策に深く関与する議員

### Tier 2（標準リサーチ）
- 当選3回以上の中堅〜ベテラン議員
- 委員長経験者
- メディア露出が多い議員

### Tier 3（最低限の基本情報）
- 当選1〜2回の若手議員
- 比例復活・比例単独の議員

## 実行上の注意

- 衆議院465名＋参議院248名＝**計713名**が対象。大規模タスクのため、政党単位でバッチ処理する
- 1政党の処理が完了するごとに `git add -A && git commit && git push` を実行する
- コミットメッセージ: `ドキュメント: {院}・{政党名}の議員プロファイル{N}名を作成`
