# ランニングコスト見積もり: 居酒屋1店舗あたり

updated_at: 2026-03-16

## 前提条件

| 項目 | 設定値 |
|------|--------|
| 店舗タイプ | 居酒屋（席数30〜50席） |
| カメラ台数 | 3台（ホール2台 + 入口1台） |
| 営業時間 | 17:00〜25:00（8時間/日） |
| 営業日数 | 月25日 |
| カメラ解像度 | 1080p / H.265圧縮 |
| フレームレート | 推論用: 5fps、録画用: 15fps |
| AI推論対象 | 人物検出・追跡（YOLO + ByteTrack）、滞在時間計測、席稼働率 |

---

## アーキテクチャ比較: 3パターン

### パターンA: フルエッジ処理（推奨）

店舗内のエッジデバイスで全推論を完結させ、クラウドには分析結果（メタデータ）のみ送信する。

```
[IPカメラ x3] → [Jetson Orin Nano] → [メタデータのみ] → [クラウド（ダッシュボード+DB）]
                  ↑ YOLO推論                              ↑ 結果表示・蓄積
                  ↑ トラッキング
                  ↑ 滞在時間計算
```

#### 月額ランニングコスト（1店舗）

| カテゴリ | 項目 | 単価 | 数量 | 月額コスト |
|----------|------|------|------|-----------|
| **エッジデバイス** | Jetson Orin Nano 電気代 | 8W × 8h × 25日 | 1台 | 約50円 |
| **カメラ** | IPカメラ電気代 | 5W × 8h × 25日 × 3台 | 3台 | 約90円 |
| **通信** | メタデータ送信（〜100MB/月） | インターネット回線に含む | - | 0円（既存回線利用） |
| **クラウド: DB** | Supabase Free / Firebase Spark | 無料枠内 | 1 | 0円 |
| **クラウド: DB（有料時）** | AWS RDS db.t3.micro (PostgreSQL) | $0.018/h | 730h | 約2,000円（$13） |
| **クラウド: ダッシュボード** | Vercel Pro | $20/月 | 全店舗共通 | 3,000円（按分） |
| **クラウド: ストレージ** | S3（分析結果JSONのみ） | $0.023/GB | 〜1GB | 約4円 |
| **クラウド: アラート** | AWS SNS / Slack Webhook | 無料枠内 | - | 0円 |

**月額合計: 約2,200〜5,200円/店舗**
- 無料DB利用時: **約2,200円**
- AWS RDS利用時: **約5,200円**

### パターンB: フルクラウド処理

映像をクラウドに送信し、クラウド上のGPUインスタンスで推論を行う。

```
[IPカメラ x3] → [映像ストリーム] → [AWS Kinesis Video Streams] → [EC2 g4dn.xlarge (T4 GPU)]
                                                                    ↑ YOLO推論
                                                                    → [RDS + S3 + ダッシュボード]
```

#### 月額ランニングコスト（1店舗）

| カテゴリ | 項目 | 計算根拠 | 月額コスト |
|----------|------|----------|-----------|
| **映像転送** | Kinesis Video Streams 取り込み | 3台 × 2Mbps × 8h × 25日 ≒ 540GB × $0.0085/GB | 約700円（$4.6） |
| **映像保存** | Kinesis ストレージ（7日保持） | 540GB × 7/30 × $0.023/GB | 約43円 |
| **GPU推論** | EC2 g4dn.xlarge | $0.526/h × 8h × 25日 | 約16,000円（$105） |
| **DB** | RDS db.t3.micro | $0.018/h × 730h | 約2,000円 |
| **ダッシュボード** | Vercel Pro | 按分 | 約3,000円 |
| **通信（上り）** | 店舗→AWS | 540GB × ISP料金 | 既存回線に含む（帯域注意） |

**月額合計: 約21,700円/店舗**

#### 注意点
- GPU常時起動が必要（営業時間のみ起動でも$105/月）
- 映像をクラウドに送信するため**プライバシーリスク大**
- 上り帯域 2Mbps × 3台 = 6Mbps が常時必要（店舗のネット回線を圧迫）
- **複数店舗で1つのGPUインスタンスを共有すれば単価は下がる**（後述）

### パターンC: ハイブリッド（エッジ推論 + クラウド録画オプション）

推論はエッジ、必要時のみ映像クリップをクラウドに保存する中間構成。

```
[IPカメラ x3] → [Jetson Orin Nano] → メタデータ → [クラウドDB+ダッシュボード]
                  ↑ YOLO推論          → イベント時の映像クリップ → [S3]
```

| エッジデバイス電気代 | Jetson + カメラ | 約140円 |

| ダッシュボード | Vercel Pro（按分） | 約3,000円 |

| **月額合計** | | **約5,200円** |

## YOLO推論の処理性能と実現可能性

### Jetson Orin Nano での YOLO 推論性能

| モデル | 精度モード | 推論時間/フレーム | FPS |
|--------|-----------|------------------|-----|
| YOLOv8n | INT8 | 23ms | 約43fps |
| YOLOv8n | FP16 | 27ms | 約37fps |
| YOLOv8s | INT8 | 28ms | 約35fps |
| YOLOv8s | FP32 | 43ms | 約23fps |

**3台のカメラを5fpsで処理する場合**: 3台 × 5fps = 15fps が必要。YOLOv8n INT8なら43fps処理可能なので**余裕あり**。

### 消費電力

| 構成 | 消費電力 |
|------|---------|
| Jetson Orin Nano（推論時） | 7.4〜8.7W |
| 1推論あたりのエネルギー | 0.13〜0.22 mWh |

1日8時間稼働で約70Wh = 約0.07kWh。電気代は約2円/日。

## 推奨構成と総コストまとめ

### 推奨: パターンA（フルエッジ処理）

| 項目 | 費用 |
|------|------|
| **初期費用** | **約105,000〜155,000円/店舗** |
| **月額ランニング** | **約2,200〜5,200円/店舗** |
| **年間ランニング** | **約26,400〜62,400円/店舗** |

### 推奨理由

1. **圧倒的な低コスト**: クラウドGPU不要のため、ランニングコストが1/4〜1/10
2. **プライバシー保護**: 映像がクラウドに出ないため、個人情報保護法上のリスクが小さい
3. **低遅延**: エッジでリアルタイム推論するため、滞在アラートの即時発報が可能
4. **帯域不要**: 店舗のネット回線を圧迫しない
5. **スケールメリット**: クラウド側は按分コストのみでスケール時にさらに安価になる

### パターンA採用時の損益分岐イメージ

月額課金を **15,000〜30,000円/店舗** と想定した場合:

| 項目 | 金額 |
|------|------|
| 売上（月額課金） | 15,000〜30,000円 |
| ランニングコスト | ▲2,200〜5,200円 |
| **粗利** | **約10,000〜28,000円（粗利率67〜93%）** |
| 初期費用回収 | 4〜15ヶ月 |
