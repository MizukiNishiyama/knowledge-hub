# 事業相談資料: 広告視聴×くじ型エンゲージメント機能の共同開発

**提案先**: 株式会社オズビジョン 代表取締役 鈴木 良 様
**提案日**: 2026年3月
**提案者**: Outarc

---

## 1. エグゼクティブサマリー

前澤じゃんけん（arigatobank）が2025年9月に休止して以来、「広告視聴 × くじ・ガチャ」型のエンゲージメントサービスには大きな空白が生まれています。

前澤じゃんけんは前澤友作氏の知名度（Twitterフォロワー1,000万人）で広告費ゼロの集客に成功しましたが、**継続的な利用を促す仕組みが弱く**、前澤氏個人への依存度が高すぎたことが休止の一因です。

一方、ハピタスは**570万人の会員基盤**、**ランク制度・宝くじなどの継続利用の仕組み**、**3,800社超の広告主ネットワーク**を持ちながら、ゲーミフィケーション要素が競合（モッピー、ポイントタウン等）と比較して手薄な状態です。

**提案の骨子**: 前澤じゃんけん型の「広告視聴 → くじ/ガチャ」機能をハピタスの新規エンゲージメント施策として共同開発し、DAUの向上とユーザーLTVの最大化を図る。

## 2. 前澤じゃんけんの分析 — 成功要因と失敗要因

### 成功した点

| 要素 | 詳細 |
|---|---|
| 爆発的集客力 | 前澤氏のSNS（1,000万フォロワー）で広告宣伝費ゼロの集客 |
| 広告収益モデル | 敗北→広告視聴→再チャレンジのループで1日最大10回の広告インプレッション |
| 射幸性の設計 | 1円〜1,000万円の賞金テーブル。15連勝（1/32,768）で夢の高額賞金 |
| メディエーション最適化 | AnyManagerの採用で広告運用コストを従来の1/12に削減 |

### 失敗した（不足していた）点

| 課題 | 詳細 | ハピタスなら解決可能？ |
|---|---|---|
| **継続利用の仕組みがない** | ランク制度・ログインボーナス・累積報酬なし。1回勝って終わりの体験 | ✅ ハピタスのランク制度・宝くじ・スタンプラリーの仕組みが応用可 |
| **前澤氏への依存** | 集客もブランドも個人に依存。持続不能 | ✅ ハピタス570万人の既存会員基盤で解消 |
| **マネタイズの多角化不足** | 広告収入＋カード発行手数料のみ | ✅ EC送客・成果報酬・OZASP連携で多角化可能 |
| **確率操作への不信感** | ブラックボックスな抽選に「操作されている」批判が多発 | ✅ 確率公開・抽選ログ保存で透明性を担保 |
| **ユーザーデータの活用不足** | 広告視聴以外のデータ蓄積・活用がなかった | ✅ ハピタスの購買データとの統合で価値向上 |

## 3. なぜオズビジョン/ハピタスか — 5つの適合性

### ① 570万人の会員基盤（CACゼロで開始可能）

前澤じゃんけんの最大の強みは前澤氏の知名度による無料集客でしたが、同時に最大の弱みでもありました。ハピタスは既に570万人の会員を持ち、月間3,000万PVのトラフィックがあります。新機能としてリリースすれば、**追加の広告費ゼロ**でDAU向上に寄与します。

### ② 継続利用の仕組みが既に存在する

ハピタスが持つ継続利用促進の仕組みは、前澤じゃんけんに欠けていた要素そのものです：

| ハピタスの仕組み | くじ機能への応用 |
|---|---|
| 4段階ランク制度（+1%〜+5%ボーナス） | ランクが高いほどくじの当選確率UP or 1日の参加回数が増加 |
| ハピタス宝くじ（毎月抽選） | くじ参加で追加の宝くじ交換券を付与（二重のインセンティブ） |
| スタンプラリー（初回7日間） | くじ機能のオンボーディングに活用し、初期定着率を向上 |
| 友達紹介（最大40%ダウン報酬） | 紹介者のくじ当選で紹介元にもボーナス → バイラル係数の向上 |

### ③ 広告主ネットワーク（3,800社超）を活用した「スポンサードくじ」

単なる広告動画視聴ではなく、ハピタスの広告主がくじの「スポンサー」になるモデルが可能です：

```
通常のハピタス案件:
  ユーザーがEC経由で購入 → 広告主が成果報酬 → ハピタスが還元

スポンサードくじ:
  ユーザーが広告動画を視聴 → くじを引く → 当選するとその広告主のクーポン/ポイント
  → ユーザーがクーポンで購入 → 通常のハピタスポイントも獲得
```

**広告主にとっての価値**: 動画広告の完全視聴が保証され、くじ当選時のブランド好感度が高い。LINEラッキーくじと類似のCRM施策だが、ハピタスなら570万人の購買意欲の高いユーザーにリーチ可能。

### ④ OZASPとの連携で独自の広告案件を確保

完全招待制のクローズドASP「OZASP」を持つオズビジョンだからこそ、くじの賞品として独自案件（高還元クレジットカード、限定EC割引等）を組み込めます。他のくじサービスでは実現できない差別化要素です。

### ⑤ 購買データとの統合で新たな価値を創出

ハピタスは年間流通総額1,790億円のEC購買データを持ちます。くじの参加行動データを購買データと統合することで：

- **広告主向け**: くじ視聴→購買のコンバージョン分析を提供（2025年12月開始の新リサーチサービスの強化）
- **ユーザー向け**: 購買履歴に基づくパーソナライズくじ（よく利用するカテゴリの賞品が当たりやすい）

## 4. 提案する機能: 「ハピタスくじ」（仮称）

### コンセプト

> ハピタスに毎日アクセスする理由を増やす「広告視聴型デイリーくじ」

### 基本フロー

```
1. ハピタスにログイン
2. 「今日のくじ」をタップ
3. スポンサー企業の動画広告を15-30秒視聴
4. くじ（じゃんけん/スクラッチ/ガチャ等）を引く
5. 当選: ハピタスポイント or スポンサー企業のクーポン
6. 1日最大5-10回参加可能（ランクに応じて増加）
```

### 賞品テーブル（案）

| 等級 | 賞品 | 確率（目安） |
|---|---|---|
| ハズレ | なし（翌日の参加権+1） | 50% |
| 7等 | 1pt | 30% |
| 6等 | 5pt | 10% |
| 5等 | 10pt | 5% |
| 4等 | 50pt | 3% |
| 3等 | 100pt | 1.5% |
| 2等 | 500pt | 0.4% |
| 1等 | 1,000pt | 0.09% |
| 特等 | 10,000pt | 0.01% |

**ポイント**: 前澤じゃんけんとは異なり、少額でも「当たる体験」を頻繁にさせることで継続利用を促す。確率テーブルは完全公開。

### ランク連動（既存制度との統合）

| ランク | 1日の参加回数 | 当選確率ブースト |
|---|---|---|
| 一般 | 3回 | なし |
| ブロンズ | 5回 | +5% |
| シルバー | 7回 | +10% |
| ゴールド | 10回 | +20% |

→ くじのためにランクを上げたい → ランクを上げるにはハピタス案件の利用が必要 → **本業の案件利用が促進される**好循環を設計。

## 5. 収益シミュレーション

### 前提条件

| 項目 | 想定値 | 根拠 |
|---|---|---|
| くじ利用DAU | 50万人（会員570万の約9%） | ハピタス宝くじの既存利用率を参考 |
| 1人1日あたり広告視聴回数 | 5回 | 1回あたり15-30秒の動画リワード広告 |
| 動画リワード広告eCPM | 2,000円（$13相当） | 日本市場の中央値。ハピタスの高属性ユーザーなら上振れ余地あり |
| ユーザー還元率 | 30% | 広告収益の30%をポイント還元 |

### 月次P/L試算

| 項目 | 計算 | 金額 |
|---|---|---|
| 月間広告インプレッション | 50万人 × 5回/日 × 30日 | 7,500万imp |
| 月間広告収益 | 7,500万imp × eCPM 2,000円 / 1,000 | **1億5,000万円** |
| ユーザー還元（30%） | | △4,500万円 |
| インフラ・開発コスト | | △1,500万円 |
| **月間粗利** | | **約9,000万円** |
| **年間粗利** | | **約10.8億円** |

### スポンサードくじの追加収益（上振れ要因）

- 広告主がくじのスポンサーとして賞品原資を負担する場合、プラットフォーム手数料10-20%を追加で取得可能
- 月10社 × 平均100万円の賞品原資 × 手数料15% = 月間150万円の追加収益
- 加えて、くじ当選→クーポン利用→EC購入の導線で、通常のハピタス成果報酬も発生

## 6. 法的整理

| 論点 | 結論 | 根拠 |
|---|---|---|
| 景品表示法 | **リスク低** | ユーザーが商品購入等の対価を支払わない「オープン懸賞」に該当。2006年に上限額規制が撤廃済み |
| 賭博法 | **該当なし** | ユーザーが参加に金銭を支払わないため「財産上の利益の得喪を争う」要件を満たさない。広告視聴は「財産上の利益」とは見なされない |
| 確率表示 | **推奨** | ソシャゲのガチャ確率表示義務化の流れに沿い、自主的に確率テーブルを全公開。信頼性の担保 |

※正式なリリース前に弁護士への確認を推奨

## 7. 競合優位性の整理

| 競合 | 弱点 | ハピタスくじの優位性 |
|---|---|---|
| 前澤じゃんけん（休止中） | 個人依存、継続利用の仕組みなし、確率不透明 | 組織運営、ランク連動、確率公開 |
| PayPayくじ | 当選額が1円〜数円、PayPay経済圏に閉じている | 高額当選もあり、ポイント交換先が多様 |
| LINEラッキーくじ | キャンペーン期間限定、LINE依存 | 常設サービス、独立プラットフォーム |
| モッピー（ガチャ機能あり） | くじ特化ではなく付帯機能止まり | 広告視聴 × くじの専用設計、スポンサードくじ |
| トリマ（ガチャ機能あり） | 移動がトリガーで室内利用に不向き | ハピタス利用の延長線上でシームレス |

## 9. 協業の形態（案）

| 形態 | 概要 | メリット | リスク |
|---|---|---|---|
| **A. 業務委託** | Outarcがくじ機能の企画・開発を受託 | シンプル。オズビジョンのリスク最小 | Outarc側の事業としてのアップサイドがない |
| **B. レベニューシェア** | くじ機能の広告収益を分配（例: OZ 70% / Outarc 30%） | 双方にインセンティブ。成果連動 | 収益配分の合意が必要 |
| **C. JV（合弁）** | くじ事業を別法人として共同運営 | 独立した意思決定、外部資金調達も可能 | ガバナンスが複雑 |

**推奨: B案（レベニューシェア）** — 初期はMVP開発をOutarcが主導し、収益が立ち始めた段階で配分を確定する段階的アプローチ。
