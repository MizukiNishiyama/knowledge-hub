# 事業アイディア管理 リファレンス

## 概要

`ideas/data/` 配下に事業アイディアごとのディレクトリを作成し、メモ書きをベースにClaude Codeがリサーチ・深掘りを行う。

## ディレクトリ構成

```
ideas/
├── reference.md          （本ファイル）
└── data/
    ├── 000_サンプルアイディア/
    │   ├── memo.md           （必須: 初期メモ・着想）
    │   ├── questions.md      （自動生成: 事業明確化のための問い）
    │   ├── research.md       （自動生成: 競合・市場調査）
    │   └── summary.md        （自動生成: 現状の整理・ネクストアクション）
    ├── 001_次のアイディア/
    │   └── ...
    └── ...
```

## 命名規則

- ディレクトリ名: `NNN_アイディア名`（NNN は 000 始まりの3桁連番）
- アイディア名は短く本質を表す名前にする（日本語可）

## ファイル仕様

### memo.md（人間が書く）

アイディアの出発点。箇条書きでも文章でも自由形式。以下の情報があるとリサーチの質が上がる。

```markdown
# アイディア名

## 着想
- どんなサービス/プロダクトか
- なぜやりたいと思ったか

## ターゲット
- 誰の、どんな課題を解決するか

## 初期仮説
- どうやってマネタイズするか
- なぜ自分がやるべきか

## メモ
- 思いついたことを自由に記述
```

### questions.md（Claude Code が生成）

memo.md を読み取り、事業を明確化するための問いを生成する。

```markdown
# 事業明確化のための問い

generated_at: YYYY-MM-DD
source: memo.md

## 顧客・課題
- [ ] ターゲット顧客は具体的に誰か？ペルソナを描けるか？
- [ ] ...

## 提供価値
- [ ] 既存の代替手段と比較して何が10倍良いか？
- [ ] ...

## ビジネスモデル
- [ ] 課金モデルは何か？（SaaS / マーケットプレイス / 広告 / etc）
- [ ] ...

## 実現可能性
- [ ] MVP に必要な最小スコープは何か？
- [ ] ...

## リスク
- [ ] 最大の不確実性は何か？
- [ ] ...
```

- チェックボックス形式にし、回答済みの問いは `[x]` に更新する
- 回答は問いの直下にインデントして記述する

### research.md（Claude Code が生成）

競合調査・市場調査の結果をまとめる。

```markdown
# リサーチ結果

generated_at: YYYY-MM-DD
source: memo.md

## 競合サービス

### [サービス名1]
- URL: https://...
- 概要: 一文での説明
- ターゲット: 誰向けか
- 価格帯: 料金モデル
- 強み:
- 弱み:
- 参考になる点:

### [サービス名2]
- ...

## 市場規模・トレンド
- TAM/SAM/SOM の推定（データソース付き）
- 関連する市場トレンド

## 技術的な実現手段
- 使えそうな技術スタック・API・SaaS
- 既存OSSやプラットフォーム

## 参考リンク
- [タイトル](URL) - 一文の説明
```

### summary.md（Claude Code が生成）

現時点での整理とネクストアクションをまとめる。

```markdown
# サマリー

updated_at: YYYY-MM-DD

## アイディア概要（1-2文）
...

## 現状の評価
- 魅力度: ★☆☆☆☆〜★★★★★
- 実現可能性: ★☆☆☆☆〜★★★★★
- 差別化: ★☆☆☆☆〜★★★★★

## 未解決の論点
- ...

## ネクストアクション
- [ ] ...
```

## Claude Code への指示テンプレート

### 新規アイディア作成

```
ideas/data/ に新しいアイディアディレクトリを作成して。
名前: 〇〇〇
以下がメモ:
---
（メモ内容をここに貼る）
---
```

### 問いの生成

```
ideas/data/NNN_アイディア名/memo.md を読んで questions.md を生成して。
```

### 競合・市場調査

```
ideas/data/NNN_アイディア名/memo.md を読んで research.md を生成して。
競合は最低5社、市場規模の推定も含めて。
```

### サマリー更新

```
ideas/data/NNN_アイディア名/ の全ファイルを読んで summary.md を更新して。
```

### 特定の問いへの深掘り

```
ideas/data/NNN_アイディア名/questions.md の「〇〇」について調査して、
結果を research.md に追記して。
```

## 運用フロー

```
1. 人間: memo.md を書く（or Claude Codeにディレクトリ作成を依頼）
2. Claude Code: questions.md を生成
3. 人間: questions.md を読み、回答を記入 or 追加の問いを依頼
4. Claude Code: research.md を生成（競合・市場調査）
5. Claude Code: summary.md を生成（現状整理）
6. 繰り返し: 問いへの回答 → リサーチ追加 → サマリー更新
```

## 注意事項

- memo.md は人間が書く唯一のファイル。他は全て Claude Code が生成・更新する
- 既存ファイルを更新する場合は上書きではなく、日付付きで追記する形式を推奨
- research.md の情報は Web 検索ベースのため、必ず裏取りすること
- generated_at / updated_at を必ず記録し、情報の鮮度を追跡可能にする
