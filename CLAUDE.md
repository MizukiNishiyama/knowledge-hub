# プロジェクトルール

## Git運用ルール

作業を完了したら、毎回以下を必ず実行すること。

1. `git add -A`
2. `git commit -m "<ラベル>: <日本語メッセージ>"`
3. `git push origin main`

### コミットメッセージ規約

- 形式: `<ラベル>: <内容を日本語で簡潔に>`
- ラベルは必ず先頭につける。
- ラベル一覧:
  - `修正`
  - `新規機能`
  - `リファクタ`
  - `ドキュメント`
  - `設定`
  - `テスト`
  - `運用`

#### 例

- `修正: books配下をknowledge-hubで一元管理するよう変更`
- `新規機能: Kindle取り込みのメタデータ出力を追加`
- `ドキュメント: セットアップ手順を日本語で追記`
