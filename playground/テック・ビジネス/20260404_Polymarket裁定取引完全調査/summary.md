## 4. リサーチ結果

### 論点1: Polymarketの市場構造はどのような価格非効率を生み出しており、それはなぜ持続するのか？

#### ファクト

**市場規模・流動性（2024-2025）**
- 2024年取引量: 約$90億。2025年取引量: 約$215億（前年比+139%）[1]
- 2025年10月の米大統領選関連のみで$33億超の出来高を記録 [1]
- 2026年2月の月次取引量: $70億（過去最高水準）[1]
- アクティブトレーダー数: 2026年初頭時点で450,000人超 [1]
- TVL（ロック総額）: 約$4.45億、オープンインタレスト: 約$4.77億 [1]
- Polymarketは予測市場全体の約45〜50%のシェアを占める（KalshiとPolymarketで85-90%） [1]

**技術構造**
- PolygonブロックチェーンにデプロイされたCLOB（Central Limit Order Book）方式 [2]
- 決済通貨はUSDC.e（BridgedUSDC）のみ [2]
- スマートコントラクトはGnosisCTF（Conditional Token Framework）+ CTFExchangeの2層構造 [3]
- アウトカムトークンはERC-1155規格のYES/NOトークン。各ペアは常に$1.00 USDC.eで完全担保 [3]
- 注文署名はEIP-712 typed data方式。オーダーブック更新はオフチェーン処理、決済のみオンチェーン [3]
- 注文成立からPoligon上での確定まで5〜12秒のレイテンシーウィンドウが存在 [2]

**価格非効率が生まれる構造的要因**
- オラクル（UMA）ベースの市場設定値が投資家の集合知（センチメント）で動くため、同一事象でも市場ごとに価格が乖離する [4]
- 従来のスポーツブックと異なり、Polymarketの賭け率は胴元が設定するのではなく市場参加者が形成するため、系統的なミスプライスが発生しやすい [4]
- 市場解決（Resolution）は7日間の「異議申し立て期間」を経てUMAオラクルが決定する。この間に表現の曖昧さや解釈の違いが生じる [5]

#### 分析・考察
Polymarketの技術設計そのものが価格非効率の温床になっている。CLOBでありながらポリゴンのブロック確定レイテンシーが存在し、かつアウトカムトークンがオンチェーンで完全担保されているため「YES + NO = $1」という理論的均衡から一時的に乖離する瞬間が繰り返し生まれる。さらに、市場解決がUMAオラクルの集合投票に依存するため、同一事象をPolymarketとKalshiが異なる基準で解決するリスクも裁定の機会であると同時にリスクでもある。

#### 小括
取引量の急拡大と450,000人を超えるトレーダーにもかかわらず、技術構造上の制約（レイテンシー・オラクル設計・市場参加者のセンチメント主導の価格形成）が価格非効率を持続的に生み出している。ただし競争の激化により機会の持続時間は短縮傾向にある。

### 論点2: 裁定取引の主要3手法のうち、2025-2026年時点で現実的に収益化できるのはどれか？

#### ファクト

**手法A: 同一市場内裁定（YES + NO < $1.00）**
- 原理: 一つの市場で YES + NO の合計が$1.00を下回る場合、両方購入すれば解決時に確実に$1.00受取 → 無リスク利益
- 実例: YES=$0.517、NO=$0.449（合計$0.966）の15分間のウィンドウで$58.52（ROI 3.4%）の利益確定 [6]
- 2024年4月〜2025年4月の実績: 同一市場内（Market Rebalancing）で$28.99M、単一条件裁定で$10.58M、計$39.57M超を上位アービトラージャーが抽出 [4]
- トップアービトラージャー: 4,049件のトランザクションで$2.01M（1件平均$496）の利益 [4]
- 機会の持続時間: 2024年は平均12.3秒 → 2026年Q1には平均2.7秒に短縮 [7]
- 利益率: 取引ごと0.5%〜2%が典型的 [6]

**手法B: クロスマーケット裁定（Polymarket vs Kalshi等）**
- 原理: 同一事象について異なるプラットフォーム間でYES/NOを買い分け、合計$1.00未満なら利益確定
- 月次リターン12〜20%を達成できるという報告あり（ただし信頼性要確認） [8]
- 実際には手数料差異（Polymarket: taker 0.3%、Kalshi: taker最大3%）が利益を圧迫 [9]
- 「スプレッドが6%超でないと手数料後に利益が出にくい」という実務経験則 [8]
- 2024年の政府機関停止事案: PolymarketがYES解決、KalshiがNO解決という真逆の解決事例が発生。クロス裁定の重大リスクを証明 [8]
- KalshiとPolymarketで市場シェアが85-90%に集中しているため、同一事象への対応力は高い [1]

**手法C: レイテンシー裁定（情報差の活用）**
- 原理: Binance等の現物取引所のBTC価格とPolymarketの5分間BTC上下予測市場の価格更新ラグ（2〜10秒）を利用
- 2024年: $51.5万/月規模の収益を上げたボット事例が報告されていた [7]
- 2025年時点: 競合ボットの増加と新動的手数料体系の導入により、ラグを利用した利益はほぼ消失。「手数料だけで裁定マージンを超える」状態に [7]
- 2026年Q1: 裁定利益の73%はサブ100ms実行ボットが独占 [7]

**手法D: コンビナトリアル裁定（複合条件の論理的不整合）**
- 原理: 「A AND B」市場と「A」「B」単体市場の価格が論理的整合性を欠く場合に利益化
- IMDEA研究では$95K程度の抽出（全体の0.24%）にとどまる [4]
- 論理的には興味深いが、スケールが小さく実用性は限定的

#### 分析・考察
2024年時点では同一市場内裁定とレイテンシー裁定が主要な収益源だったが、2025-2026年では競争環境が激変している。特にレイテンシー裁定は2025年1月の動的手数料体系導入（50/50確率時でtaker最大1.56%）により採算が悪化。一方で同一市場内裁定は今も$39M超の実績を持つ最大の機会だが、機会の持続時間が12.3秒から2.7秒へ短縮されており、サブ100msのインフラなしでは競争できない状況になりつつある。クロスマーケット裁定は解決リスクを許容できる高スプレッド時（6%超）に限れば依然として有効。

#### 小括
2026年時点で現実的なのは「同一市場内裁定（高頻度・小利益の積み上げ）」と「クロスマーケット裁定（スプレッド6%超限定）」の組み合わせ。レイテンシー裁定は手数料構造の変化により事実上死んでいる。いずれも高速インフラと相当な競争への対処が必要。

### 論点3: 裁定ボットを実際に構築・運用するために必要な技術スタック・コードパターン・インフラは何か？

#### ファクト

**公式SDK・API**
- 公式Python SDK: `py-clob-client`（PyPI: `pip install py-clob-client`、Python 3.9+） [10]
- ホスト: `https://clob.polymarket.com`、Polygon Chain ID: 137
- WebSocket: リアルタイムオーダーブック更新・約定イベント・市場解決のストリーミング対応 [2]
- Gamma API: 市場データ（メタデータ・結果・流動性）取得用のREST API [2]
- Data API: 過去トレードデータ・オーダーブックスナップショット取得用 [2]

**認証・注文署名の仕組み**
```python
from py_clob_client.client import ClobClient

HOST = "https://clob.polymarket.com"
CHAIN_ID = 137  # Polygon mainnet

client = ClobClient(
    HOST,
    key=PRIVATE_KEY,      # ウォレット秘密鍵
    chain_id=CHAIN_ID,
    signature_type=1,     # Magic/emailウォレット用
    funder=FUNDER_ADDRESS
)
client.set_api_creds(client.create_or_derive_api_creds())
```
注文はEIP-712署名付き構造化データとして送信。Polymarketのリレイヤーがオンチェーンガス代を肩代わりする（ガスレス設計） [2][10]

**指値注文・成行注文の実装例**
```python
from py_clob_client.clob_types import OrderArgs, MarketOrderArgs, OrderType
from py_clob_client.order_builder.constants import BUY

# 指値注文（GTC: Good-Till-Cancelled）
order_args = OrderArgs(
    price=0.45,
    size=100.0,   # USDC建て
    side=BUY,
    token_id="<YES_TOKEN_ID>"
)
signed_order = client.create_order(order_args)
resp = client.post_order(signed_order, OrderType.GTC)

# 成行注文（FOK: Fill-Or-Kill）- 裁定に適したタイプ
mo = MarketOrderArgs(
    token_id="<NO_TOKEN_ID>",
    amount=100.0,
    side=BUY
)
signed = client.create_market_order(mo)
resp = client.post_order(signed, OrderType.FOK)
```

**裁定機会検出のコアロジック**
```python
def detect_arbitrage(yes_price, no_price, min_roi=0.02):
    """同一市場内裁定の検出"""
    total_cost = yes_price + no_price
    if total_cost < 1.00:
        profit_per_dollar = 1.00 - total_cost
        roi = profit_per_dollar / total_cost
        if roi >= min_roi:  # 最低ROI 2%以上
            return {
                "opportunity": True,
                "roi": roi,
                "profit_per_dollar": profit_per_dollar
            }
    return {"opportunity": False}
```

**推奨インフラ要件（競争力のある実装のために）**
- 注文署名: EIP-712をRustかGoで実装すると100-300ms短縮可能 [2]
- WebSocket受信: 10ms以内
- ローカルオーダーブック処理: 1ms以内
- 合計目標レイテンシー: サブ100ms
- 同時監視市場数: 100市場以上
- 推奨資本: 1条件あたり$10K〜$50K [6]
- 最低ROI閾値: 2%以上（ガス・手数料考慮後）

#### 分析・考察
公式SDKとAPIは十分に整備されており、Pythonで素早くプロトタイプを作れる環境は整っている。ただし本格的な競争力を持つには署名処理をRust/Goに移行し、サブ100msのレイテンシーを実現する必要がある。ImMike等のOSSリポジトリは学習・プロトタイピングに有用だが、実運用では競合上位ボットに対して遅れを取る可能性が高い。

#### 小括
基本実装はPythonと公式SDKで数日で構築可能。ただし収益化できるかどうかは技術よりもレイテンシーと資本規模の競争になっており、実運用ではRust/Goへの移行と十分な資本が必要条件になっている。

### 論点4: リスク・コスト・規制上の制限を考慮したとき、期待リターンはポジティブになりうるか？

#### ファクト

**コスト構造**
- Polygonガス代: 通常1トランザクションあたり$0.01〜$0.50。Polymarketのリレイヤーが代払い（実質ガスレス）のため、通常は意識不要 [9]
- MATIC保有推奨量: 直接対話時に0.1〜1 MATIC（数十円〜数百円）で数千トランザクション分 [9]
- Taker手数料: 2026年1月導入の動的手数料。50/50確率時に最大1.56%（15分暗号市場では最大1.80%）、確率が極端（0.05/$0.95近辺）では0%に近づく [9]
- Maker手数料: 0%（ただし通常はMaker rebateあり: 約0.20%還元） [9]
- Kalshi手数料（比較）: Taker最大3%、一部市場でMaker手数料もあり [8]
- 往復手数料（概算）: 両サイドtakerの場合$0.60〜3%のコストが発生

**レゾリューションリスク（最大の非線形リスク）**
- 2024年政府機関停止問題: PolymarketとKalshiが同一事象に対して真逆の解決（YES vs NO）を下した実例あり [8]
- 2025年6月Zelenskyスーツ事案: $2.4億市場でUMAオラクルの大口トークン保有者（投票権の25%）による影響が疑われる解決 [5]
- UMAオラクルは「オプティミスティック」設計のため、最終解決まで7日間の異議申し立て期間が存在 [5]
- 解決失敗時は全額返金（draw）されるケースもあるが、その間資金がロックされる

**流動性制約**
- 2025年調査: 流動性の低い市場では裁定機会の78%が実行非効率で失敗 [5]
- オーダーブックの深さが浅い市場では$500超のポジションでスリッページが顕著
- 高流動性市場（選挙等）は機会が多いが競争も激烈

**規制状況（日本居住者の観点）**
- Polymarketは2022年に米国ユーザーを全面禁止（CFTC規制違反の和解）。2025年12月にCFTC承認下でPolymarket USAとして招待制で再参入 [5]
- 日本居住者: 現時点では明示的な禁止はなく、利用可能。ただし国内の金融商品取引法上の位置づけは明確でなく、グレーゾーン [5]
- KYC: 国際版Polymarketはウォレット接続のみで可能（KYC不要）。US版は完全KYC義務化 [5]
- 禁止国・地域: 米国（国際版）、北朝鮮、イラン、シリア、キューバ等 [5]

**競争環境の変化**
- ICEがPolymarketに$20億投資（評価額$80億、2025年10月）。機関投資家の本格参入が始まっている [7]
- 上位10アービトラージャーが全利益の21%を独占。スケールと速度の競争 [4]
- 「初期暗号裁定のように、機関投資家が本格参入すると構造的非効率は圧縮される」という専門家見解 [7]

#### 分析・考察
コスト面では、Polygonのガスレス設計により取引コストは低く抑えられているが、2026年1月から導入された動的Taker手数料（最大1.80%）が従来無料だったモデルから大きく変わった点は重要。ROI 2-3%の薄利スプレッドを狙う戦略では往復手数料だけで採算を失いうる。最大のリスクはレゾリューションリスクで、Polymarket-Kalshi間クロスポジションは特に真逆解決の可能性を常に内包している。規制については日本居住者には現時点で大きな障壁はないが、グレーゾーン認識は必要。

#### 小括
期待リターンはポジティブになりうるが、以下の条件が必須：(1) Taker手数料を上回るスプレッドの確保（実質4-5%以上）、(2) レゾリューション発散リスクを考慮したクロスマーケット裁定の慎重な判断、(3) 流動性リスクを回避できる市場選択。競争の激化により2026年以降は機会の質と量が低下する見通しであり、早期参入・継続的な戦略更新が前提条件となる。

## 5. まとめ

### 総合所見
Polymarketの裁定取引は実在する収益機会であり、学術研究により2024年4月〜2025年4月の1年間で$39.6M超が抽出されたことが実証されている。しかし、競争の激化・動的手数料体系の導入・機会持続時間の短縮（12.3秒→2.7秒）により、「誰でも作れるボットで稼げる」時代はほぼ終わっており、現在はサブ100msのインフラと十分な資本を持つ高度なプレイヤーの世界になりつつある。今から参入するなら、クロスマーケット裁定の「高スプレッド機会（6%超）の選択的実行」が最も現実的なアプローチである。

### 確度の高い結論
- 同一市場内裁定（YES + NO < $1.00）は理論的に無リスクであり、2025年も$39M超の実績が学術論文で確認されている
- 公式Python SDK（py-clob-client）で2-3日以内にプロトタイプ実装は可能
- Polygonガス代は実質無料（リレイヤー代払い）であり、コスト上のボトルネックではない
- 2026年1月以降の動的Taker手数料（最大1.80%）により、薄利スプレッド戦略の採算が著しく悪化した
- クロスマーケット裁定は解決基準の相違（実例あり）により、表面上のスプレッドに反して損失が確定するリスクが存在する

### まだ不確実な点
- ICE等機関投資家の本格参入後に市場非効率がどのペースで圧縮されるか
- 日本居住者に対して将来的に規制が強化されるかどうか
- Polymarket USAの成功可否が国際版のルール変更（KYC義務化等）に波及するか
- UMAオラクルのガバナンス改善により解決リスクが低下するかどうか

### ネクストアクション
