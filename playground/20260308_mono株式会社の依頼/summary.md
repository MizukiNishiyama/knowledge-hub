## 2. 問いの明確化

### 背景・文脈
MONO株式会社は「one-fanz.jp」という全国のサッカー強豪校向けクローズド動画配信プラットフォームを運営している。保護者・チーム・選手が試合や練習試合をいつでも閲覧できるサービス（月額770円/ユーザー）であり、ここに「選手別動画の自動生成」と「ハイライトの自動抽出」の2機能を追加したい。Veo（カメラ+AI処理の統合サービス）がほぼ網羅的に実現しているが、MONO社は既存の配信基盤を活かした独自ソフトウェアとして構築したい意向。打ち合わせ（3/10 15時）までに技術的な可能性・期間・コスト感を検討する必要がある。

### 核心的な関心事
- 選手識別・追跡→選手別動画の自動生成は、現在の技術でどこまで実現可能か？精度・コスト・開発期間はどの程度か？
- ハイライトの自動抽出は、どのような技術アプローチが現実的か？既存サービスやOSSモデルで使えるものは何か？
- ローカル処理とクラウド処理、それぞれの実現性とコスト構造は？

### 暗黙の前提・仮説
- 撮影はVeoのような専用カメラではなく、既存のカメラ（固定カメラ等）で撮影された動画を後処理する想定
- one-fanz.jp は既に動画配信インフラを持っているため、AI処理部分の追加開発が焦点
- ユースサッカーの試合映像が対象であり、プロリーグ水準の複数アングルカメラは期待できない
- 「選手識別」は背番号ベースでの識別が前提と推測される

## 4. リサーチ結果

### 論点1: 選手の識別・追跡を実現する技術的アプローチとして何が現実的か？

#### ファクト

**物体検出＋追跡（Detection + Tracking）: 技術的に成熟**
- **YOLOv8/v11 + ByteTrack/BoT-SORT** が現在の標準的なパイプライン [1][2]
- ByteTrack: MOT17ベンチマークで MOTA 80.3、IDF1 77.3。ニューラルネットワーク不使用の純粋なアルゴリズムで高速（30 FPS） [3]
- BoT-SORT: MOT17/MOT20で1位（MOTA 80.5、IDF1 80.2、HOTA 65.0）。カメラモーション補償機能あり [4]
- YOLOv8（medium）による選手検出精度: 適切なデータで mAP 87.9%以上（Nature Scientific Reports掲載の研究） [5]
- **YOLO26**（2026年1月リリース）: YOLO11比でCPU推論43%高速化、NMSフリー推論対応 [6]

**Roboflow supervision ライブラリ（@skalskip92 = Piotr Skalski が主要開発者）**
- GitHub 36,600+ スター、MIT ライセンス [7]
- サッカー特化のパイプラインを提供: 選手検出 → チーム分類 → カメラキャリブレーション → 2Dレーダー可視化 [8]
- **チーム分類**: SigLIP（画像特徴量抽出）+ UMAP（次元削減）+ K-means で2チーム+審判を自動分類 [8]
- **カメラキャリブレーション**: YOLOv8x-pose で32個のピッチ特徴点を検出し、ホモグラフィ変換でピクセル座標→実座標にマッピング [9]
- **roboflow/sports** リポジトリ（4,900+ スター）: ボール追跡、背番号認識、選手追跡、Re-ID、カメラキャリブレーションの5課題に対応 [10]

**背番号認識（Jersey Number Recognition）**
- CVPR 2025 Workshop での最新研究: 不確実性モデリング + PARSeq で SoccerNet テストセットで **87.45%** の精度 [11]
- ただしチャレンジセット（より難しい条件）では **79.31%** に低下 [11]
- Roboflow のバスケットボール実験: ResNet-32 ベースの背番号OCRが **93%** の精度を達成（ただしバスケは解像度が高い） [12]
- 課題: 低解像度、モーションブラー、背番号が見えない角度で精度が大幅に低下 [11]
- キーフレーム選択（背番号が最も見えるフレームの自動選択）で精度が約37%向上する研究あり [13]

**選手Re-ID（再同定）**
- Sports Re-ID 論文: 階層的サンプリング + セントロイド損失で mAP +7〜11.5ポイント向上 [14]
- 同一チームの選手はユニフォームが類似するため、一般的なRe-IDより難易度が高い [14]
- **SoccerNet Game State Reconstruction (GSR)**: 選手追跡+個人識別+ピッチ上位置推定のEnd-to-Endタスク。200シーケンス、236万の選手位置アノテーション [15]
- **日本からの参加**: MIXI + Playbox が CVPR 2025 SoccerNet GSR チャレンジで世界4位 [16]

**SAM2（Segment Anything Model 2）**
- Meta社のビデオセグメンテーションモデル。約44 FPSで動作 [17]
- プロンプトベース（点・ボックス・マスク指定）でフレーム間の時系列一貫性を維持 [17]
- 手動で初期フレームで選手を指定→以降を自動追跡するユースケースに最適 [17]

#### 分析・考察

選手追跡には「検出+トラッキング」と「個人識別」の2段階がある。

**検出+トラッキング（実現性: 高）**:
YOLO + ByteTrack/BoT-SORT は技術的に成熟しており、supervision ライブラリを使えば数週間で基本パイプラインを構築可能。チーム分類（SigLIP + K-means）も自動化できる。

**個人識別（実現性: 中〜低）**:
3つのアプローチがある:
1. **背番号OCR**: 最も直感的だが、ユースサッカーの固定カメラ1台では解像度・角度の問題で安定しない（最先端でも79-87%）。ただしキーフレーム選択技術で改善可能
2. **外見特徴Re-ID**: 同チーム選手のユニフォームが同じため極めて難しい
3. **SAM2を使った手動初期指定+自動追跡**: 試合開始時に手動で選手を指定し、SAM2の時系列追跡で追尾するハイブリッド方式。**現時点で最も現実的な落としどころ**

**推奨パイプライン**:
```
YOLO11 (検出) → BoT-SORT (追跡) → SigLIP+K-means (チーム分類)
→ SAM2 (手動初期指定+追跡) or 背番号OCR (自動識別)
→ supervision (可視化・レーダーマップ)
```

#### 小括
選手の検出・追跡・チーム分類は supervision + YOLO で十分実現可能（技術リスク低）。個人識別は「SAM2を活用した初期手動指定+自動追跡」のハイブリッドが現実解。背番号OCRによる完全自動化は段階的に精度を向上させるロードマップが妥当。

### 論点2: ハイライトの自動抽出にはどのような手法があり、サッカーに最適なアプローチは何か？

#### ファクト

**商用サービス**
- **WSC Sports**: AI ハイライト自動生成の最大手。NBA、FIFA、Bundesliga、日本ではWOWOWが採用。WOWOW導入後、ハイライト視聴者が74%増加、AI生成ハイライトは手動編集の翌日版より **30倍** 多く視聴された [18][19]
- **Pixellot**: 無人カメラ+AI。80カ国以上、16スポーツ対応。Pixellot Air カメラ $849。RESTful API でハイライトを `ClipNotification` webhook で受信可能 [20][21]
- **Veo**: カメラ+AI統合。Veo Cam 3 $1,533、サブスク $42-$178/月。自動ゴール・シュートタグ付け、選手別ハイライトリール生成。40,000+チーム、100+カ国で利用 [22][23]
- **Cognitive Mill**: クラウドAIサービス。映像のみから確率的AI+教師なしクラスタリングでハイライト検出。$0.25〜/分。APIあり [24]
- **Trace**: ユースサッカー特化。選手ごとのパーソナライズド動画を自動生成。月額 $25/選手 [25]

**学術ベンチマーク・オープンソース**
- **SoccerNet Action Spotting**: サッカー映像分析の主要ベンチマーク。500試合、**17種類のアクションクラス**（ゴール、シュート、ファウル、カード等）のアノテーション付き。Dense Detection Anchors が最先端で mAP 約74.84% [26][27]
- **SoccerNet 2025 チャレンジ**（第5回）: 280名参加。Team Ball Action Spotting、Monocular Depth Estimation、Multi-View Foul Recognition、GSR の4タスク [27]
- **Lighthouse**（LINE社開発）: 動画のモーメント検索・ハイライト検出ライブラリ。7つのモデル内蔵（Moment-DETR, QD-DETR等）。Apache 2.0ライセンス。EMNLP 2024、ICASSP 2025/2026に採択 [28]
- **VideoMAE V2**: 動画の自己教師あり事前学習モデル。Kinetics-400で90.0%。ハイライト検知のバックボーンとして利用可能 [29]

**技術的アプローチ**
- **音声ベース**: 歓声・実況の音量変化でハイライトシーンを推定。メルスペクトログラム+CNN で **89%** の精度（Della Santa & Lalli, 2025）。ただしユースサッカーでは実況がないため適用制限あり [30]
- **ボール追跡ベース**: ボールのゴールライン通過、急激な方向変化でゴール・シュートを検出。supervision + ボール検出モデルで実装可能 [8]
- **動きベース（Optical Flow）**: 選手の急激な動き変化、ゴール後の集合パターンを検出 [30]
- **マルチモーダル融合**: 音声+映像の組み合わせが単一モダリティを上回る（音声89%、映像83%、組合せが最高性能）[30]
- **LLM活用**: Claude 3 Sonnet で映像フレームをマルチモーダル分析し、ハイライトシーンを特定するAWS Serverlessパイプラインの実装例あり [31]

#### 分析・考察

ユースサッカーのハイライト抽出には、以下の3段階アプローチが現実的:

**Phase 1（MVP）: ボール追跡+ルールベース**
- ボールのゴールエリア進入→ゴールライン通過 = ゴール
- プレー停止時間の長さ = 重要イベント（PK、レッドカード等）
- 選手の急な集合 = ゴール後の喜び
- 実装容易、説明可能性が高い

**Phase 2: SoccerNet モデルによるイベント分類**
- 17種類のアクションクラスを学習済みモデルで分類
- mAP 70-75%程度は期待できる
- GPU処理が必要だが、追加で15-30分程度

**Phase 3: マルチモーダル（音声があれば）**
- ユースサッカーでも保護者の歓声は拾える可能性
- 音声スパイクとボール追跡を組み合わせれば精度向上

「ハイライト」の定義を段階的に拡張するのが鍵。ゴールだけなら比較的容易（ボール追跡ベースで十分）。好プレーやチャンスシーンの検出は難易度が上がるが、SoccerNetモデルで対応可能。

#### 小括
ゴール検知（ボール追跡ベース）から始め、段階的にSoccerNet学習モデルで17種イベントに拡張するアプローチが現実的。完全自動で放送品質のハイライトを初期から目指すのは開発コストが大きい。

#### 分析・考察

競合は3カテゴリに分かれる:
1. **ハードウェア統合型**（Veo, Pixellot, NTTSportict）: 専用カメラ+AIの垂直統合。精度は高いが導入コストも高い
2. **分析特化ソフトウェア型**（SPLYZA, FL-UX, Hudl）: 既存映像を手動/半自動で分析。AI自動処理は限定的
3. **配信プラットフォーム型**（SOCCER FANz, Player!）: 動画の配信・共有が中心。分析機能は弱い

**one-fanz.jp の差別化機会**:
1. **「配信+AI分析」の統合**: 現在のone-fanz.jpは「配信プラットフォーム型」だが、ここにAI分析を載せることで、SPLYZA（分析のみ）やSOCCER FANz（配信のみ）を超えた統合サービスになれる
2. **保護者向け「うちの子のプレー」動画**: Trace の$25/選手/月モデルが参考。保護者が最も欲しい機能であり、日本では提供しているサービスがほぼない
3. **既存顧客基盤の活用**: 全国の強豪校が既に入っており、ゼロからの顧客開拓が不要
4. **価格**: SPLYZA（16,500円/月/チーム）より選手単位課金にすれば、保護者が直接負担できる（月額2,000-3,000円/選手）

**最大の脅威**: Veoの日本本格参入（既にサッカーショップKAMOで販売開始 [40]）、およびPlaybox + MIXI のAI映像分析（CVPR世界4位の技術力 [16]）

#### 小括
ハードウェア統合型（Veo等）と差別化するには「既存カメラ映像のソフトウェア処理」に特化し、「配信+AI分析」の統合サービスとして日本のユースサッカー×保護者ニーズにフォーカスするのが有効。競合上の最大の脅威はPlaybox/MIXIの技術力とVeoの日本進出。

- Vast.ai A100 で1-2時間処理: **$0.5-1.0（≒75-150円）** [42]
- ストレージ・データ転送含めて1試合あたり **$1-5（≒150-750円）** [41][42]

**ローカル処理のハードウェア要件**

| グレード | GPU | 価格 | 性能 |
|---------|-----|------|------|
| 推奨 | RTX 3060 (12GB) | 5-6万円 | YOLO medium, 60-80 FPS@1080p |
| 高性能 | RTX 4090 (24GB) | 25-30万円 | フルパイプライン, 100+ FPS |
| ミニマル | GTX 1660 (6GB) | 2-3万円 | YOLO nano, 30-50 FPS@720p |

[41][43]

**エッジ vs クラウドの損益分岐点**
- RTX 4090（30万円）÷ 1試合$3 ≒ 10万試合分のクラウドコスト
- ただし電気代・メンテ含めて **月100試合以上** を継続処理するならローカルが有利 [41]

#### 分析・考察

one-fanz.jp のユースケースでは:
- **試合数**: 全国の強豪校が対象なら月数十〜数百試合
- **リアルタイム性**: 不要（試合後にアップロードして処理する運用）
- **スケーラビリティ**: 大会期間中は試合が集中する

推奨アーキテクチャ:

| フェーズ | 処理方式 | 理由 |
|---------|---------|------|
| 初期（〜月50試合） | クラウド（Vast.ai / RunPod） | 初期投資不要、1試合$1-3で開始可能 |
| 成長期（月50-200試合） | ハイブリッド（ローカル+クラウドバースト） | RTX 4090サーバー1台+大会時はクラウドで増強 |
| 本格展開（月200試合+） | 専用サーバー複数台 | コスト最適化 |

「ローカルで処理できるもの」という先方の要望について: RTX 4090 搭載PC（約40-50万円）があれば技術的には可能。90分の試合を30-60分で処理できる。ただし24時間稼働・故障対応等の運用面を考慮すると、初期はクラウドから始める方がリスクが低い。

#### 小括
1試合あたりのクラウド処理コストは$1-5（150-750円）、処理時間は30分〜3時間（パイプライン複雑度による）。初期はクラウド処理で開始し、月100試合超でローカルGPUサーバーへの移行を検討するのが合理的。

### 論点5: 日本のユースサッカー映像分析市場の現状と、one-fanz.jpの差別化機会は何か？

#### ファクト

**日本のスポーツアナリティクス市場**
- 2025年: 約1.12億ドル（≒168億円） [44]
- 2026年予測: 約2.80億ドル [44]
- 2034年予測: 約8.45億ドル [44]
- CAGR: 25.18%（2026-2034） [44]

**高校サッカーの規模**
- 全国の高校サッカー登録チーム数: 約4,000校 [45]
- 全国大会出場レベルの強豪校: 約200-300校 [45]
- 全国高校サッカー選手権: 全47試合がTVerで無料配信されるようになり、視聴ニーズが拡大 [46]

**グローバルサッカー分析ソフトウェア市場**
- 2024年: 5.22億ドル [47]
- 2032年予測: 11.8億ドル（CAGR 12.1%） [47]

**国内映像分析の普及状況**
- **全国高校サッカー選手権出場校の約30%** が SPLYZA Teams を利用 [34]
- J-Leagueクラブの30%が FL-UX を利用 [35]
- 川崎フロンターレU-18がPwC支援でAI映像分析（自動シーン抽出、ラベリング、3D→2D鳥瞰位置マッピング）を導入 [48]
- ユース・高校レベルでの**AI自動**映像分析はほぼ未開拓。既存サービスは手動タグ付けが中心 [34][35]

**保護者のニーズ**
- 「うちの子のプレーだけを見たい」は保護者の最も強いニーズの一つ [37]
- 遠征先の試合を見に行けない保護者にとって、動画配信サービスの需要は高い [37]
- Jリーグでも選手カメラ（個人追跡映像）は人気機能 [35]

**映像分析文化の浸透**
- 「映像を観て振り返る文化」が高校サッカー部で広まりつつある [34]
- 選手主導の分析（コーチからの指示待ちではなく、選手自身が映像を分析）がトレンド [34]
- AI カメラ（Veo、NTTSportict）による撮影コスト低減で映像の供給が増加 [37][40]

#### 分析・考察

**市場の空白地帯**: 日本のユースサッカーでは、「映像の撮影・配信」と「手動での分析」は普及しているが、**AI自動処理（選手追跡、ハイライト生成）は空白**。SPLYZA は手動タグ付け、SOCCER FANz は配信のみで、AI 自動分析を提供するサービスがない。

**one-fanz.jp のポジショニング機会**:

```
            配信あり          配信なし
AI分析あり   ★one-fanz.jp     Playbox(ベータ)
             (=狙うべき位置)
AI分析なし   SOCCER FANz      SPLYZA Teams
```

**市場規模の試算**:
- 強豪校300校 × 選手25名 × 月額2,000円 = 月1,500万円（年間1.8億円）
- 全国4,000校に拡大: 年間24億円の潜在市場
- サッカー以外のスポーツ（野球、バスケ、バレー等）への展開可能性を含めると市場ポテンシャルはさらに大きい

**リスク要因**:
- Playbox + MIXI（CVPR世界4位の技術力）が同じ領域を狙っている [16][39]
- Veo が日本でサッカーショップKAMO を通じて販売開始しており、ハードウェア統合型として脅威 [40]
- NTTSportict が STADIUM TUBE Lite でアマチュアスポーツ市場に進出中 [37]

#### 小括
日本のユースサッカーAI映像分析は空白市場。one-fanz.jp は「配信基盤+AI分析」の統合で、SPLYZA（分析のみ・手動）やSOCCER FANz（配信のみ）を超えたポジションを取れる。ただしPlaybox/MIXI、Veo日本進出が競合リスク。先行者利益を取るにはスピードが重要。

## 5. まとめ

### 総合所見
選手追跡・ハイライト自動生成は、OSSエコシステム（supervision + YOLO11 + ByteTrack/BoT-SORT + SAM2）を活用すれば技術的に実現可能であり、1試合あたり$1-5（150-750円）のクラウド処理コストでビジネスモデルが成立する。日本のユースサッカーでは「配信+AI自動分析」の統合サービスが空白市場であり、one-fanz.jp の既存基盤（顧客・配信インフラ）を活かした早期参入に大きな機会がある。ただし完全自動の個人識別は精度制約（背番号OCR: 79-87%）があり、「SAM2による手動初期指定+自動追跡」のハイブリッドが現実解となる。

### 確度の高い結論
- 選手検出・追跡・チーム分類は supervision + YOLO で十分実現可能（技術的リスク低）
- 1試合あたりのクラウド処理コストは$1-5で、ビジネスモデルとして成立する水準
- 日本のユースサッカーでAI自動映像分析を提供しているサービスはほぼ存在しない（市場機会大）
- Veo が最も近い競合だが、専用カメラ購入+サブスクが必要な点で、ソフトウェア型のone-fanz.jpとは差別化可能
- ハイライトのゴール検知（ボール追跡ベース）は比較的容易に実装可能
- Roboflow/supervision エコシステムがMITライセンスで利用可能であり、開発工数を大幅に削減できる

### まだ不確実な点
- 固定カメラ1台のユースサッカー映像における背番号OCR精度（実際の映像でのテストが必要）
- ハイライト検知のゴール以外のイベント（好プレー・チャンス等）の精度
- 保護者の支払い意思額（月額2,000-3,000円/選手が受容されるか）
- Veo の日本市場本格参入のタイミングと販売戦略
- Playbox + MIXI の製品化・商用リリースの時期

### ネクストアクション
- **即座に（打ち合わせまでに）**: 以下の3段階ロードマップとコスト概算を準備する
  - **Phase 1 (MVP, 2-3ヶ月)**: YOLO11 + BoT-SORT + supervision でチーム分類付き選手追跡 + SAM2で手動指定追跡。ボール追跡ベースのゴール検知。開発チーム: ML エンジニア1-2名
  - **Phase 2 (4-6ヶ月)**: 背番号OCR による自動個人識別。SoccerNet モデルによるイベント分類拡張（17種）。クラウド処理パイプライン構築
  - **Phase 3 (6-12ヶ月)**: 完全自動化、走行距離等の統計データ生成、API/ダッシュボード、スケーラブルインフラ
- **1週間以内**: one-fanz.jp の実際の試合映像を使って supervision + YOLOv8 でPoC実施（選手検出・追跡の精度検証）
- **2週間以内**: 背番号OCRの精度を実映像でテストし、ハイブリッド方式のUX設計
- **中期**: 保護者へのヒアリング（選手別動画にいくら払えるか、どんな形式で見たいか）
