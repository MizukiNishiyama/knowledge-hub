# リサーチレポート: Difyはオワコンか

## 1. note.mdの内容

> Difyは非エンジニアでもAIエージェント・アプリがGUIで作れるサービスで、NTTDATAなどが正規代理店になるなど、日本のAISaaS界隈でそこそこのプレゼンスを持ってきた。
> しかし、このサービスはオワコンで今後のMRRは下落していくと考えている。
> 理由は「人間がかなり介入していくことを前提としているからである。」
> 競合としては、LangchainやAWS AgentCoreなど非エンジニアが触れないようなライブラリやサービスになるが、ClaudeCodeなどでかなり自然言語ベースでコーディングができるようになったため、GUIベースで人間が途中過程を見ながら開発するUXが最適ではなくなっている。
> この主張を中立に判断した上で、未来予測をせよ

---

## 2. 問いの明確化

### 背景・文脈
「Difyはオワコン」という主張は、NTT DATAパートナーシップなどの日本での台頭を見ながらも、GUIベース・人間介入前提というDifyのコアな設計思想が、Claude CodeなどのLLM-nativeな開発環境の普及によって時代遅れになるという仮説に基づいている。競合として挙げられているLangChain/AWS AgentCoreは「非エンジニアが触れないツール」であり、それらがDifyのターゲット層（非エンジニア）を奪うというシナリオが暗黙に前提されていると読み取れる。

### 核心的な関心事
- Difyの「GUI・人間介入前提」設計は、AIコーディングツールの台頭によって本当に価値を失っているか？
- Difyの今後の収益は拡大するのか、縮小するのか？その根拠は？
- 「非エンジニアがClaude Codeで開発できる時代」という前提は現実として正しいか？

### 暗黙の前提・仮説
- 「AIコーディングツール（Claude Code等）の普及 = 非エンジニアもコードを書けるようになる」という前提がある
- LangChain・AWS AgentCoreをDifyの直接競合として位置付けているが、ターゲット層が異なる可能性がある
- GUIベース開発の価値は「エンジニアがコードを書けないから」生まれるという前提がある

## 4. リサーチ結果

### 論点1: Difyの実際の成長軌跡は「オワコン」を示しているか？

#### ファクト
- 2026年3月、Difyは **$30M Series Pre-A** を調達、バリュエーション **$180M** に到達 [1]
- GitHubスター数 **131,000**（全OSS史上51位）、デプロイ機器数 **140万台**、展開国数 **175カ国以上** [1][2]
- 商用チーム **2,000+**、エンタープライズ顧客 **280社**（Maersk, Novartis, ETS等） [1]
- 収益 **$3.1M**（2025年6月時点、28名チームで達成） [3]
- 日本市場：NTT DATA「つなぎAI」がDify Enterpriseを採用、**2027年までに600社・累計売上40億円**を目標設定 [5]
- CyberAgent：社員の約20%（約1,800名）が6ヶ月以内に導入、**月3,000時間削減** [6]
- Ricoh：2025年9月時点で開発アカウント約2,000名、業務利用約4,300名 [7]
- トヨタ紡織：通常3ヶ月かかるAIアプリ開発を2週間（**約80%短縮**）で実現 [5]

#### 分析・考察
2026年3月現在、Difyはあらゆる指標で「成長中」と評価できる。$30M調達・131K GitHub Stars・日本での大企業採用（CyberAgent・Ricoh・トヨタ紡織）は縮小傾向の組織のそれではない。収益規模（$3.1M）は競合n8n（$40M ARR）と比較すると1/13程度であり、まだ初期フェーズであることは確かだが、2026年3月の調達ラウンドは投資家がその成長軌跡を評価した証左だ。「オワコン」説を支持するデータは現時点では存在しない。

#### 小括
現時点のDifyは「オワコン」どころか加速成長フェーズにある。ただし絶対収益規模は小さく、n8nとの格差拡大リスクは注視が必要。

### 論点2: Claude Code等のAIコーディングツールはDifyのコアユーザー層を代替できるか？

#### ファクト
- Difyの中核ユーザー：マーケター・営業担当者等の**ビジネス職（非エンジニア）**が主力（CyberAgent事例での実態） [6]
- Claude Codeの参入障壁：**CLIファースト、ターミナル必須、月$20〜$200**、新しいメンタルモデルが必要 [14]
- Vibe Codingツール（Lovable, Bolt）は非エンジニア向けに設計されているが、**本番環境への実際のデプロイ率は9%**（ビジュアルビルダーの65.2%に対して） [15]
- Bubble調査（793名、2025年9〜10月）：ミッションクリティカル用途への自信 ビジュアル開発 **71.5%** vs Vibe Coding **32.5%** [15]
- Stack Overflow 2025調査（65,000名）：**72%の開発者が業務でVibe Codingを使っていない** [16]
- Vibe Codingユーザーの63%が非開発者だが、業務スケールへの壁は依然大きい [17]
- Dify → Claude Code / AIコーディングツールへの乗り換え事例：**確認できない**（ツールの用途が根本的に異なるため）

#### 分析・考察
Claude Codeは「コード生成支援ツール」であり、DifyはAIアプリの「実行・構築プラットフォーム」である。ターゲット層が異なる。CyberAgentのマーケターや営業がClaude Code（ターミナル必須のCLIツール）を代替ツールとして選ぶ可能性は極めて低い。より近い競合はVibe Codingツール（Lovable/Bolt）だが、本番運用・セキュリティ・メンテナビリティの面でGUIビルダーが圧倒的に優位にある。

「ある程度技術素養がある人材（テックリードや開発者）がDifyを使っていた層」はCursor/Claude Codeへの移行可能性があるが、これはDifyの主要顧客層ではない。Difyの価値の本質は「エンジニアなしで本番品質のAIアプリを構築できる」点にある。

#### 小括
DifyのコアユーザーはClaude Codeを使えない。「GUI・人間介入前提」はDifyにとって欠点ではなく、非エンジニアが本番利用できる唯一の経路という価値の源泉そのものである。

### 論点3: LangChain/AWS AgentCoreはDifyの真の競合か？

#### ファクト
- LangChain：コードファースト・開発者専用フレームワーク、非エンジニアは使用不可。収益 **$16M**（Difyの5倍）、バリュエーション **$1.25B**（2025年10月にユニコーン到達）。2024年にDifyはLangChainをコードベースから削除し独立。 [11]
- AWS AgentCore：エンタープライズMLエンジニア向けのフルマネージドインフラ。ターゲットはClearwater Analytics, Sony, Thomson Reutersのような大企業のエンジニアチームであり、GUIビルダーではない。 [18][19]
- 真の競合：**n8n**（$40M ARR、$2.5B valuation、GitHub 70K+スター）が最大の競合 [10]
- Flowise（最も近い競合だったOSSビルダー）は2025年8月にWorkdayが買収、独立市場から退出 [12]
- n8nは400+統合の汎用自動化プラットフォームにAIを統合、DifyはAI/LLM特化GUI環境という差別化

#### 分析・考察
LangChainとAWS AgentCoreはDifyの「直接競合」ではない。ターゲット層（エンジニア vs 非エンジニア）が根本的に異なり、ユーザーの代替行動が起きにくい構造だ。LangChainとDifyはむしろ相互補完的（LangChainで開発したモデルをDify上でデプロイする使い方もある）。

真の脅威はn8nのような汎用自動化プラットフォームのAI化と、Make.com/ZapierのAI機能強化による「AIアプリ構築」領域への侵食だ。ただしFlowiseのWorkday買収はGUIビルダー独立勢の縮小を意味し、Difyにとってはむしろ追い風となっている。

note.mdの「競合 = LangChain / AWS AgentCore」という設定は競合構図の誤認である可能性が高い。

#### 小括
LangChainとAWS AgentCoreはレイヤーが異なりDifyの直接競合ではない。真の競合はn8nであり、FlowiseのWorkday買収でDifyの独立OSS競合は事実上消えた。

### 論点4: 「ノーコードGUI vs LLM-native」の競争軸において業界はどちらを選んでいるか？

#### ファクト
- ノーコードAIプラットフォーム市場規模：2025年 **$6.56B**、CAGR **31%**で2034年には **$75B** 予測 [20]
- Kenneth Auchenberg（元Microsoft・Stripe）が2025年4月に「**No Code is Dead. Long live Vibe Coding.**」を発表 — 業界で最も注目された「GUI不要論」 [22]
- 実態（Bubble調査、793名、2025年9〜10月）：本番クリティカルアプリへの実際のデプロイ率 ビジュアル **65.2%** vs Vibe Coding **9%** [15]
- 開発速度3倍以上と回答：ビジュアル開発 **77.4%** vs Vibe Coding 22% [15]
- WorkdayがFlowise（GUIビルダー）を2025年8月に買収：エンタープライズのGUIビルダー需要を評価した最も強いシグナル [12]
- 日本市場：ノーコードAI開発プラットフォーム市場 **2025年度4,085億円**（ミック経済研究所）[20]

#### 分析・考察
「No Code is Dead」論は2025年に盛り上がったが、本番環境での採用データを見ると実態は異なる。Vibe Codingはプロトタイプ・個人プロジェクトには強いが、ガバナンス・セキュリティ・メンテナビリティが求められるエンタープライズ環境ではGUIビルダーが選ばれ続けている。WorkdayによるFlowiseの買収は「エンタープライズはGUIビルダーを本格採用している」という最も強力な市場シグナルだ。

ノーコードAI市場の$75B（2034年）予測が正しければ、GUIビルダーの需要は縮小ではなく拡大する。競争軸の変化は起きているが、それは「GUIの敗北」ではなく「GUIがエンタープライズ化し、LLM-nativeが個人・スタートアップ向けに棲み分ける」という二層化と解釈するほうが整合的だ。

#### 小括
業界の現実データはGUIビルダーの本番運用での優位性を示している。「LLM-nativeがGUIを置き換える」という論は少なくとも短中期（2〜5年）では誇張。ただし10年スパンでの構造変化リスクは排除できない。

### 論点5: DifyはAIコーディングツール台頭という構造変化に対して戦略的に適応できているか？

#### ファクト
- 2024年：DifyはLangChainをコードベースから削除し、独自アーキテクチャへ完全移行 [11]
- 2025年2月：日本法人 **LangGenius K.K.** を設立（東京・日本橋）、4名体制でスタート [4]
- 2025年9月：**一般社団法人Dify協会**設立 — エコシステム構築を制度化 [23]
- 2026年3月：$30M調達と同時に「エンタープライズグレードのエージェンティックワークフロー」を強調 — エンタープライズピボットの明示 [1]
- エンタープライズ顧客数 **280社**（Maersk, Novartis, ETS等グローバル大企業） [1]
- 投資家陣容：Mizuho Leaguer Investment（みずほ銀行系）を含む日本投資家も参加 [1]

#### 分析・考察
Difyの動きは「現状維持」ではなく「エンタープライズ化」への明確なシフトとして読み取れる。日本法人設立・Dify協会設立・みずほ系投資家の参加はすべて日本エンタープライズ市場への本格参入を示している。「GUI依存 = 弱点」ではなく「エンタープライズのガバナンスニーズ = GUIの強み」という再定義が進んでいる。

ただし、n8nとの収益差（$3.1M vs $40M）は依然大きく、エンタープライズ化競争でn8nに追い付けるかは不確実だ。また中長期的にはAnthropicが「Cowork」（非技術者向けLLM-nativeファイルエージェント）をリリースし始めており、「非エンジニアでも使えるLLM-nativeツール」の出現は潜在的な脅威として残る。

#### 小括
Difyはエンタープライズ化という明確な戦略を取っており、現段階の適応力は十分。ただし中長期的なLLM-nativeツールのUI進化は引き続き注視が必要。

## 5. まとめ

### 総合所見
「DifyはオワコンでMRRは下落していく」という主張は、2026年3月現在のデータと整合しない。Difyは131K GitHub Stars・$30M調達・日本の大企業（CyberAgent・Ricoh・トヨタ紡織）での本格採用という加速成長フェーズにある。核心的な問題は、「GUI・人間介入前提」はDifyの弱点ではなく、非エンジニアが本番品質のAIアプリを構築できる唯一の経路という価値であり、競合として挙げられたLangChain/AWS AgentCore/Claude Codeはそもそもターゲット層が異なりDifyのユーザーを奪えない構造にある。

### 確度の高い結論
- **短期（2〜3年）：DifyのMRRは下落しない。** エンタープライズ展開（NTT DATA経由600社目標）により日本市場でも成長の根拠がある
- **競合構図の誤認：** LangChainとAWS AgentCoreはDifyの競合ではない。真の競合はn8nとVibe Codingツール（Lovable/Bolt）
- **Vibe Codingは本番運用でGUIビルダーを代替できていない：** 本番デプロイ率9% vs ビジュアル65.2%という実態がある
- **市場は拡大中：** ノーコードAI市場は2034年に$75B予測（CAGR 31%）であり、縮小トレンドにはない

### まだ不確実な点
- **中長期（5〜10年）のリスク：** AnthropicのCoworkのような「非エンジニア向けLLM-native UI」が成熟したとき、GUIビルダーの価値が侵食されるリスクは残る
- **n8nとの競争：** n8nはDifyの13倍の収益・14倍のバリュエーション。エンタープライズ競争でDifyが追い付けるかは未知数
- **AIガバナンス規制の強化：** 市民開発者（Citizen Developer）の裁量を制限する組織ポリシーが広がると、Difyのユーザー層が縮小する可能性

### ネクストアクション
- Dify活用を検討するなら：NTT DATA「つなぎAI」の実際の採用企業の評判をヒアリングし、ROI実態を確認する
- Dify vs n8nの選定をするなら：AI特化ユースケース → Dify、汎用業務自動化 + AI → n8n という軸で評価する
- 長期リスクを評価するなら：Anthropic Cowork・Lovableの本番環境採用率を6ヶ月後に再調査する
