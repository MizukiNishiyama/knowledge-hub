# Playground リファレンス

## 概要

`playground/{directory_name}/note.md` に記載されたメモ・問い・関心事に基づき、プロのリサーチャーとして徹底的な調査・分析・資料作成を行う。

## ディレクトリ規約

- 作業完了後、ディレクトリ名を `{YYYYMMDD_name}` 形式にリネームする
  - 例: `my_topic` → `20260308_my_topic`
- すべての成果物は `playground/{カテゴリ}/{YYYYMMDD_name}/` に保存する
- カテゴリは下記「カテゴリ分類ルール」に従い、AIが自動で判定する

---

## カテゴリ分類ルール

`playground/` 直下には以下の6カテゴリのディレクトリを設置する。
リサーチ完了後、`{YYYYMMDD_name}` ディレクトリを **必ず** 該当カテゴリ配下に配置すること。

| カテゴリ | 対象トピック | 判断基準の例 |
|---------|------------|------------|
| `政治` | 政府・政策・選挙・政治家・法律・国際政治 | 「〜政策」「〜議員」「政府の対応」「選挙」など |
| `テック・ビジネス` | AI・プロダクト・スタートアップ・市場・事業戦略・ツール | 「〜のビジネス」「〜市場」「AI/LLM」「SaaS」など |
| `人物` | 特定の人物の経歴・発言・思想・影響力のリサーチ | 「〜の思想」「〜について」「人物名が主題」など |
| `カルチャー` | 音楽・映画・アート・文学・サブカルチャー・エンタメ | 「アーティスト名」「作品名」「文化的コンテンツ」など |
| `思想` | イデオロギー・哲学・世界観・価値観のマッピング | 「〜思想マッピング」「〜主義」「哲学」「価値観」など |
| `その他` | 上記いずれにも明確に当てはまらないもの | 判断に迷った場合はここに分類し、後で移動する |

### 分類手順（/play 実行時）

1. note.md のテーマを読み、上記テーブルに照らして最も適切なカテゴリを1つ選ぶ
2. 保存先パスを `playground/{カテゴリ}/{YYYYMMDD_name}/` とする
3. ディレクトリが存在しない場合は作成する（カテゴリ自体は既に存在するため通常不要）
4. summary.md・source.md・summary.pdf をそのパスに保存する
5. ディレクトリのリネーム（`mv`）もカテゴリ配下で行う

## 成果物仕様

### 1. summary.md

以下の6セクション構成で作成する。各セクションの品質基準を厳守すること。

---

#### セクション1: note.mdの内容

- note.md の原文をブロック引用（`>`）でそのまま掲載する
- 一切の編集・要約を加えない

---

#### セクション2: 問いの明確化

**目的**: note.md の散発的なメモを、調査可能な形に構造化する

**品質基準**:
- note.md に書かれた意図・背景・文脈を読み取り、「この人は本当は何を知りたいのか」を言語化する
- 暗黙の前提や仮説があれば明示する
- 曖昧な表現は具体的な問いに変換する

**フォーマット**:
```markdown
## 2. 問いの明確化

### 背景・文脈
（note.md から読み取れる背景を2-3文で整理）

### 核心的な関心事
- （最も知りたいことを1-3個、明確な問いの形で）

### 暗黙の前提・仮説
- （note.md の記述から推定される前提を列挙）
```

---

#### セクション3: 論点設定

**目的**: セクション2の問いを、リサーチ可能な具体的論点に分解する

**品質基準**:
- 3〜6個の論点を設定する（多すぎず少なすぎず）
- 各論点は「Yes/Noで答えられない、調査と考察を要する問い」にする
- 表面的な事実確認だけでなく、「So what?（だから何？）」まで踏み込む問いを含める
- 論点間に論理的な流れ（現状把握→構造理解→将来展望など）を持たせる

**悪い論点の例**:
- 「〇〇とは何か？」（単なる定義の確認）
- 「〇〇の市場規模は？」（数字を調べるだけ）

**良い論点の例**:
- 「〇〇が急成長している構造的要因は何か？その持続性は？」
- 「既存プレイヤーが〇〇に取り組めていない理由は何か？それは解消されうるか？」
- 「〇〇の普及における最大のボトルネックは何か？どのような条件で解消されるか？」

**フォーマット**:
```markdown
## 3. 論点設定

| # | 論点 | 調査の方向性 |
|---|------|-------------|
| 1 | （問い） | （どのようにアプローチするか） |
| 2 | （問い） | （どのようにアプローチするか） |
| ... | ... | ... |
```

---

#### セクション4: リサーチ結果

**目的**: 各論点に対して、根拠ある回答を提示する

**品質基準**:
- 各論点につき最低3つ以上のソースを参照する
- 事実（ファクト）と解釈（分析）を明確に区別する
- 数値データがある場合は必ず含める（市場規模、成長率、ユーザー数など）
- 相反する見解がある場合は両論併記する
- 「わからなかった」「情報が見つからなかった」も正直に記載する

**フォーマット（論点ごとに繰り返し）**:
```markdown
## 4. リサーチ結果

### 論点1: （論点をそのまま再掲）

#### ファクト
- （調査で判明した事実。ソース番号を付記 [1][2]）
- （数値データがあれば明記）

#### 分析・考察
（ファクトを踏まえた解釈。複数の見方がある場合は併記）

#### 小括
（この論点に対する暫定的な回答を1-2文で）

---

### 論点2: ...
```

---

#### セクション5: まとめ

**目的**: リサーチ全体を総合し、actionableな示唆を導く

**品質基準**:
- 単なる要約ではなく、論点横断的な洞察を含める
- 「結局どういうことか」を3文以内で言い切る
- 確度の高い結論と、まだ不確実な部分を区別する
- 具体的なネクストアクションを提示する

**フォーマット**:
```markdown
## 5. まとめ

### 総合所見
（リサーチ全体を通じた結論を3文以内で）

### 確度の高い結論
- （根拠が十分にある結論）

### まだ不確実な点
- （追加調査が必要な点）

### ネクストアクション
- （具体的に次にやるべきこと）
```

---

#### セクション6: 追加の論点

**目的**: リサーチを踏まえて新たに浮上した問いを提示し、思考を拡張する

**品質基準**:
- リサーチ前には見えなかった、調査を経て初めて立てられる問いにする
- 「だったら〇〇はどうなるのか？」という連鎖的な問いを含める
- 実務的に重要な問いと、知的好奇心を刺激する問いの両方を含める
- 2〜4個程度

**フォーマット**:
```markdown
## 6. 追加の論点

| # | 追加論点 | なぜこの問いが重要か |
|---|---------|-------------------|
| 1 | （問い） | （この問いを追うべき理由） |
| ... | ... | ... |
```

### 2. summary.pdf

summary.md を PDF に変換し、同じディレクトリに保存する。

**変換方法**:
`markdown` と `WeasyPrint` の2ライブラリを使用する。

```python
import markdown
from weasyprint import HTML

# 1. Markdown → HTML
with open("summary.md", "r") as f:
    md_text = f.read()

html_body = markdown.markdown(md_text, extensions=["tables", "fenced_code"])

# 2. HTML を整形（日本語フォント・余白・テーブルスタイル指定）
html_full = f"""<!DOCTYPE html>
<html><head><meta charset="utf-8">
<style>
  body {{ font-family: "Hiragino Sans", "Noto Sans CJK JP", sans-serif; font-size: 11pt; line-height: 1.7; margin: 40px; }}
  h1 {{ font-size: 18pt; border-bottom: 2px solid #333; padding-bottom: 4px; }}
  h2 {{ font-size: 15pt; border-bottom: 1px solid #999; padding-bottom: 3px; margin-top: 28px; }}
  h3 {{ font-size: 13pt; margin-top: 20px; }}
  h4 {{ font-size: 11pt; margin-top: 16px; }}
  table {{ border-collapse: collapse; width: 100%; margin: 12px 0; font-size: 10pt; }}
  th, td {{ border: 1px solid #ccc; padding: 6px 10px; text-align: left; }}
  th {{ background: #f5f5f5; font-weight: bold; }}
  blockquote {{ border-left: 4px solid #ccc; margin: 12px 0; padding: 8px 16px; color: #555; }}
  code {{ background: #f4f4f4; padding: 2px 4px; border-radius: 3px; font-size: 10pt; }}
  pre {{ background: #f4f4f4; padding: 12px; border-radius: 4px; overflow-x: auto; }}
</style>
</head><body>{html_body}</body></html>"""

# 3. HTML → PDF
HTML(string=html_full).write_pdf("summary.pdf")
```

**注意事項**:
- `pip install markdown weasyprint` が必要
- 日本語フォントが環境にインストールされていること（macOS は Hiragino Sans が標準搭載）
- テーブル・コードブロック・引用が正しくレンダリングされることを確認する

### 3. source.md

## Few-shot Example

以下は、note.md の内容が「日本のポッドキャスト市場って伸びてるの？参入余地ある？」だった場合の出力例（抜粋）。

---

### summary.md の例（抜粋）

```markdown
# リサーチレポート: 日本のポッドキャスト市場

## 1. note.mdの内容

> 日本のポッドキャスト市場って伸びてるの？参入余地ある？

## 2. 問いの明確化

### 背景・文脈
ポッドキャスト市場への参入を検討している。市場の成長性と、後発でも勝てる余地があるかを見極めたい。

### 核心的な関心事
- 日本のポッドキャスト市場は今後も成長するか？その根拠は？
- 新規参入者が差別化できるポジションは残っているか？

### 暗黙の前提・仮説
- 米国ではポッドキャストが大きく成長しており、日本も追随するはずという前提がある
- 「参入」がコンテンツ制作なのかプラットフォームなのかは未定義

## 3. 論点設定

| # | 論点 | 調査の方向性 |
|---|------|-------------|
| 1 | 日本のポッドキャスト市場が米国ほど伸びていない構造的要因は何か？ | 米日比較、通勤習慣、言語圏の規模、広告市場構造 |
| 2 | 既存プレイヤー（Spotify, Apple, Voicy等）のポジショニングと未充足ニーズは？ | 各社の戦略分析、ユーザーレビュー、不満点の調査 |
| 3 | マネタイズモデルとして何が現実的か？広告以外の収益源は？ | サブスク、投げ銭、BtoB活用、教育コンテンツ等 |
| 4 | 生成AI・音声合成の進化がポッドキャスト市場に与える影響は？ | AI音声の品質動向、コスト構造の変化、新しいフォーマットの可能性 |

## 4. リサーチ結果

### 論点1: 日本のポッドキャスト市場が米国ほど伸びていない構造的要因は何か？

#### ファクト
- 米国のポッドキャストリスナーは人口の42%（約1.4億人）に対し、日本は15.7%（約1,980万人）[1][2]
- 日本の音声広告市場は2025年に420億円と推定されるが、ポッドキャスト広告はそのうち約50億円程度 [3]
- 日本の通勤時間の中央値は片道48分で、音声コンテンツ消費の余地はある [4]

#### 分析・考察
米国との差は「習慣の壁」が大きい。日本では通勤中のコンテンツ消費は動画（YouTube）やSNSが主流であり、音声専用コンテンツへの移行が進んでいない。一方で、ながら聴きの価値が浸透すれば急速にキャッチアップする可能性もある。

#### 小括
市場は成長途上にあるが、米国型の急成長を前提にするのは危険。日本固有の消費習慣に適応した戦略が必要。

（以下、論点2〜4も同様の構造で続く）

## 5. まとめ

### 総合所見
日本のポッドキャスト市場は成長しているが、米国の5年遅れではなく構造的に異なる市場と捉えるべき。参入余地はあるが、米国の成功モデルの直輸入ではなく、日本の音声消費習慣に最適化したアプローチが鍵。

### 確度の高い結論
- リスナー数は年15-20%で増加しており、縮小リスクは低い
- 広告モデル単体での収益化は日本市場では困難

### まだ不確実な点
- 生成AIによる制作コスト低下が市場にどう影響するか
- BtoBポッドキャスト（社内研修・IR等）の潜在需要の規模

### ネクストアクション
- ターゲットセグメント（BtoC vs BtoB）を先に決定する
- 日本のポッドキャストヘビーユーザー10名にインタビューを実施し、未充足ニーズを特定する

## 6. 追加の論点

| # | 追加論点 | なぜこの問いが重要か |
|---|---------|-------------------|
| 1 | YouTubeのポッドキャスト機能強化は脅威か味方か？ | Googleの参入が市場拡大に寄与するか、独立系を潰すかで戦略が変わる |
| 2 | 日本語話者の海外リスナー（在外日本人）は有望なニッチか？ | 国内市場だけでなく、言語圏全体で捉えると見える機会がある |
| 3 | ショート動画世代（Z世代）は長尺音声コンテンツを聴くのか？ | 将来の主要リスナー層の行動パターンが市場の天井を決める |
```

## リサーチの姿勢

### やるべきこと
- **一次情報を優先する**: 公式発表、統計データ、決算資料、論文を重視する。個人ブログやまとめ記事は補助的に使う
- **数字で語る**: 「成長している」ではなく「年率18%で成長」。定量データを必ず探す
- **反証を探す**: 自分の仮説に都合の良い情報だけでなく、反する情報も積極的に探す
- **「なぜ？」を繰り返す**: 表面的な事実の裏にある構造・メカニズムまで掘り下げる
- **日本語と英語の両方で検索する**: 日本固有のテーマでも英語圏に有益な分析があることが多い
- **最新情報を重視する**: 2年以上前のデータは、より新しいデータがないか確認する

### やってはいけないこと
- ソースなしで断定する
- 1つの情報源だけで結論を出す
- note.md に書かれていない前提を勝手に追加する
- 「一般的に」「広く知られている」など曖昧な表現で根拠を省略する
- 情報が見つからない論点を無視する（「見つからなかった」と明記する）
