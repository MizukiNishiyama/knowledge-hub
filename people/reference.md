# people/reference.md

## 目的
人物を「自分の関心軸」に沿って構造化し、
学習・意思決定・戦略設計に活用するためのリファレンス設計。

Cloud Code / Codex で人物名 `{name}` が指定された場合：
1. ウェブサーチを行う
2. 本フォーマットに基づいて情報整理する
3. `people/{name}/summary.md` として保存する
4. 出典情報を `people/{name}/sources.md` に保存する

## ディレクトリ構成

```
people/
├── reference.md              （本ファイル）
└── {name}/
    ├── summary.md            （自動生成: 人物情報サマリー）
    └── sources.md            （自動生成: 出典・参考リンク）
```

## 4. 概要（30文字以内）
端的に何者かを表す一文。

## 5. 詳細（100文字以内）
経歴・現在の活動・代表的実績を簡潔に。

## 6. 行動原理（実現したい社会・信念）
- どのような社会を目指しているか
- どのような思想・哲学に基づいて行動しているか
- 意思決定の根本原理は何か

抽象→具体の順で記載。

## 7. 興味深い主張
その人物を特徴づける主張・発言・論点。
最大5項目。
可能であれば出典URLを sources.md に保存。

## 8. 参考にできる点
以下の観点から抽出する：
- 再現可能な意思決定フレーム

- 情報の信頼性が弱い

# 「自分の趣味・思考」を反映するための評価軸（運用者プロファイル）
このプロジェクトは、単なる人物要約ではなく「自分の関心に沿ってリサーチし、学びを抽出する」ためのもの。
よって出力では、以下の観点を必ず拾いに行く（該当が無い場合は “該当なし” と明記）。

### 関心クラスタ（優先順）
- 生成AI/LLM、AIエージェント、RAG、OCR、MCP、プロダクト化
- ガバナンス：責任・監査・ログ・制度設計（国家/行政/企業統治）
- 事業：SIの限界、DX、エンタープライズ、M&A/Exit、バリュエーション/資本政策
- 思想：社会構造の設計、ポストAGIの世界観、古代中国思想（統治/秩序/倫理）
- カルチャー：音楽（特にHIPHOP/クラブ文脈）、スポーツ分析

### 参考抽出のルール（「参考にできる点」生成指針）
- “再現可能な行動/意思決定フレーム” に落とす  
  例：意思決定基準、組織設計、学習法、交渉・資本政策の作法、制度設計の哲学
- “実装” を意識して箇条書き（3〜7個）
- 自分の文脈（起業/AI/ガバナンス/財務/政治志向）に接続して書く

### 「参考にすべきではない点」生成指針
- 生存者バイアス（運・時代要因が大きい）
- 倫理/法務/社会コストの無視
- スケールしない個人芸
- 情報ソースが弱い主張（陰謀論・未検証・誇張）

## ウェブサーチ規約（最低限）
- 可能なら一次情報を最低1つ入れる（公式サイト/論文/登壇/本人SNSなど）
- 重要な主張は “どこで言ったか” を sources.md に紐付け
- 不確実な点は断定しない（「〜と報じられている」「本人の〜発言による」）

# 出力制約

- 概要：30文字以内
- 詳細：100文字以内
- 主張は簡潔に
- 断定しすぎない
- 必ず出典を保存する
- 再現可能性を重視する
